# 配置下发与同步 — 功能需求说明书

## 1. 背景与价值

**为什么做**：配置下发与同步是连接平台和网关的关键模块，负责将设备实例的采集配置部署到边缘网关（Node-RED），并在配置变更时同步更新。没有配置下发，设备实例无法在网关上运行，整个系统无法采集数据。

**做成什么样**：用户点击下发后，平台自动生成 Node-RED Flow 并部署到网关，网关开始采集数据并上报平台。全程自动化，无需登录网关操作。

**关联**：依赖于边缘网关管理和设备实例管理模块。

---

## 2. 范围

### 本次做

- 平台手动添加网关后的心跳 flow 自动下发
- 心跳流下发失败时网关状态为未激活
- 设备实例采集 flow 手动下发（单个 + 批量）
- 配置版本号与已下发版本号比对
- 全量重新部署实例 flow
- 批量下发按网关分组并行，同网关内串行
- 实例删除时调用 Node-RED API 删除对应 flow
- 删除时如果 flow 不存在则跳过并记录
- 下发历史记录
- 下发状态与失败原因展示
- 采集数据通过 MQTT 上报平台

### 本次不做

- 网关离线时排队等待下发
- 自动下发设备实例采集 flow
- 增量更新 flow
- 网关侧配置人工修改的检测与冲突合并
- 用户权限与角色管理

## 4. 需求详述

### 4.1 核心流程

> 这里只描述**主线路径的全局视角**——用户从哪里开始、经过哪些关键节点、到哪里结束。

**心跳 flow 自动下发流程**
1. 用户在平台手动添加网关，填写 Node-RED 地址、端口、Basic Auth 认证信息
2. 网关注册成功，系统自动下发心跳上报 flow
3. 下发成功 → 网关状态变为"已激活"
4. 下发失败 → 网关创建成功，但状态为"未激活"

**采集 flow 下发流程**
1. 用户创建设备实例，绑定网关，配置通信参数和点位
2. 用户点击下发配置
3. 系统比对配置版本号和已下发版本号
4. 有变化则全量下发，无变化则提示跳过
5. 系统生成 Node-RED Flow 并部署
6. Flow 命名：`设备实例名称-设备ID`
7. 下发成功记录历史，失败记录原因

**批量下发流程**
1. 用户选择多个实例
2. 系统按网关分组
3. 不同网关并行下发
4. 同网关内串行下发
5. 每条记录单独显示结果

### 4.2 交互规则

- 删除操作需二次确认
- 下发失败弹窗显示具体失败原因
- 网关离线时下发操作给予明确提示
- 批量下发时每条记录显示独立结果

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 每次实例配置变化产生新配置版本号，下发成功后更新已下发版本号 |
| BR-002 | 每个实例对应一个独立 flow，命名规则为`设备实例名称-设备ID` |
| BR-003 | 批量下发按网关分组并行，同网关内串行 |
| BR-004 | 使用网关配置中的 Basic Auth 调用 Node-RED API |
| BR-005 | 采集数据通过 MQTT 发布到平台侧 Broker，再写入时序库 |

## 5. 验收标准

> AC 是这份文档的核心交付物。后续的技术方案、任务拆分、TDD 测试用例全部从这里派生。

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 心跳 flow 自动下发
  - **Given** 用户在平台手动添加网关
  - **When** 网关创建成功
  - **Then** 系统自动下发心跳上报 flow

- [ ] **AC-002**: 心跳 flow 下发成功激活网关
  - **Given** 心跳上报 flow 下发成功
  - **When** 下发完成
  - **Then** 网关状态变为"已激活"

- [ ] **AC-003**: 生成并部署采集 flow
  - **Given** 设备实例已绑定网关
  - **When** 用户点击下发配置
  - **Then** 系统为该实例生成独立 flow 并部署到绑定网关的 Node-RED

- [ ] **AC-004**: Flow 命名规则
  - **Given** 设备实例已绑定网关
  - **When** 生成 flow
  - **Then** flow 名称为"设备实例名称-设备ID"

- [ ] **AC-005**: 有变化才下发
  - **Given** 实例配置版本号大于已下发版本号
  - **When** 用户点击下发配置
  - **Then** 系统全量重新下发该实例 flow

- [ ] **AC-006**: 无变化提示跳过
  - **Given** 实例配置版本号等于已下发版本号
  - **When** 用户点击下发配置
  - **Then** 提示"无需下发"

- [ ] **AC-007**: 不同网关并行下发
  - **Given** 用户选择多个实例批量下发
  - **When** 实例分布在不同网关
  - **Then** 系统按网关分组并行下发，不同网关互不阻塞

- [ ] **AC-008**: 同网关串行下发
  - **Given** 用户选择多个实例批量下发
  - **When** 多个实例属于同一网关
  - **Then** 系统在该网关内串行下发

- [ ] **AC-009**: 下发成功记录历史
  - **Given** 下发配置成功
  - **When** 下发完成
  - **Then** 记录下发历史，状态为成功，并更新该实例的已下发版本号

- [ ] **AC-010**: 删除实例清理 flow
  - **Given** 用户删除设备实例
  - **When** 该实例 flow 存在于网关 Node-RED
  - **Then** 系统调用 Node-RED API 删除对应 flow，并删除平台实例记录

- [ ] **AC-011**: 下发历史展示
  - **Given** 下发历史列表
  - **When** 查看历史
  - **Then** 每条记录显示实例名称、设备ID、网关名称、下发版本号、下发类型、状态、失败原因、下发时间、操作人

- [ ] **AC-012**: 采集数据上报
  - **Given** 采集数据流正常运行
  - **When** Node-RED 采集到数据
  - **Then** 数据通过 MQTT 发布到平台侧 Broker，平台接收后写入时序库

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-013**: 心跳 flow 下发失败
  - **Given** 平台手动添加网关
  - **When** 心跳上报 flow 下发失败
  - **Then** 网关创建成功但状态为"未激活"，并记录失败原因

- [ ] **AC-014**: 下发时网关离线
  - **Given** 用户点击下发配置
  - **When** 绑定网关离线
  - **Then** 下发失败，提示"网关离线，下发失败"

- [ ] **AC-015**: 下发失败后可重试
  - **Given** 下发失败
  - **When** 用户修复配置后
  - **Then** 用户可手动重新下发

- [ ] **AC-016**: 删除时 flow 不存在
  - **Given** 用户删除设备实例
  - **When** 网关上对应 flow 不存在
  - **Then** 平台删除实例记录，并记录"网关配置不存在，跳过删除 flow"

- [ ] **AC-017**: 全量覆盖被修改的 flow
  - **Given** 网关上的 flow 被人工修改
  - **When** 用户重新下发配置
  - **Then** 平台以自身配置为准，全量覆盖对应实例 flow

- [ ] **AC-018**: 批量下发部分失败
  - **Given** 批量下发中某个实例失败
  - **When** 后续还有其他实例待下发
  - **Then** 同网关内后续实例继续执行，并在结果中标记失败项

- [ ] **AC-019**: 删除实例网关离线
  - **Given** 用户删除设备实例
  - **When** 网关离线
  - **Then** 仍删除平台实例记录，并记录"网关离线，未删除对应 flow"

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-020**: 版本号递增 ← BR-001
  - **Given** 设备实例配置变更
  - **Then** 实例配置版本号递增；下发成功后，已下发版本号更新为当前配置版本号

- [ ] **AC-021**: 一个实例一个 flow ← BR-002
  - **Given** 设备实例配置下发
  - **Then** 每个实例对应一个独立 flow

- [ ] **AC-022**: Flow 命名规则 ← BR-002
  - **Given** Flow 命名
  - **Then** 实例 flow 名称固定为"设备实例名称-设备ID"

- [ ] **AC-023**: 批量下发分组规则 ← BR-003
  - **Given** 批量下发
  - **Then** 按网关分组并行，同一网关内串行

- [ ] **AC-024**: Flow 部署目标 ← BR-004
  - **Given** 下发目标
  - **Then** 设备实例 flow 部署到该实例绑定的网关 Node-RED 上

- [ ] **AC-025**: Node-RED 认证 ← BR-004
  - **Given** Node-RED 认证
  - **Then** 使用网关配置中的 Basic Auth 认证信息调用 Node-RED API

- [ ] **AC-026**: 数据上报 ← BR-005
  - **Given** 采集数据上报
  - **Then** 采集数据统一通过 MQTT 发布到平台侧 Broker，再写入时序库
