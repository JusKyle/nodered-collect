# 配置下发与同步 - 页面功能交互说明

## 1. 页面综述

- **功能描述**：管理设备实例采集配置的下发与同步，支持心跳 flow 自动下发、采集 flow 手动下发（单个+批量）、配置版本比对、下发历史记录和状态追踪。
- **页面类型**：后台管理类 (Admin / Table-First)
- **目标用户**：工业自动化工程师、IoT 系统运维人员、数据采集系统管理员
- **核心目标**：用户可以在平台上完成采集配置的下发、状态追踪和历史查看，无需登录网关操作。
- **本阶段范围**：心跳 flow 自动下发、采集 flow 手动下发（单个+批量）、配置版本比对、下发历史记录、实例删除自动清理 flow

---

## 2. 页面结构

- **页面标题**：配置记录
- **页面副标题**：查看所有配置下发历史和状态
- **主体布局**：导航栏 + 顶部筛选区 + 数据表格区
- **主要页面/区域**：
  - 导航栏：采集网关，注册码，设备模型，设备实例，配置记录，系统配置
  - 配置记录列表页（主页面）
  - 配置记录详情页（二级页面）
  - 下发配置（操作入口在设备实例页）
  - 批量下发（操作入口在设备实例页）
  - 下发失败详情（弹窗）
  - 删除确认（气泡）

---

## 3. 配置记录列表页

### 3.1 搜索/筛选区

- **搜索字段**：
  - `实例名称`：输入框，默认空，模糊搜索
  - `设备ID`：输入框，默认空，模糊搜索
- **筛选字段**：
  - `网关名称`：下拉选，默认`全部`，选项为已注册网关列表
  - `下发类型`：下拉选，默认`全部`，选项：`心跳流`、`采集流`
  - `状态`：下拉选，默认`全部`，选项：`成功`、`失败`
- **时间范围筛选**：
  - `下发时间`：日期范围选择器，默认空
- **交互**：
  - 输入搜索关键词后需点击`查询`按钮
  - 选择筛选条件后需点击`查询`按钮
  - 搜索与筛选条件叠加生效
  - 点击`重置`按钮清空所有筛选条件，恢复默认状态

### 3.2 操作按钮区

- **`刷新`**：次要按钮，点击重新加载列表数据

### 3.3 列表展示区

- **排序规则**：默认按 `下发时间` 倒序排列。
- **列表字段**：
  - `实例名称`：设备实例名称，加粗显示，可点击跳转到设备实例详情页
  - `设备ID`：设备唯一标识，无数据显示 `-`
  - `网关名称`：绑定的网关名称，可点击跳转到网关详情页
  - `下发版本号`：本次下发的配置版本号，无数据显示 `-`
  - `下发类型`：使用标签展示，映射关系：
    - `心跳流` → 蓝色标签
    - `采集流` → 绿色标签
  - `状态`：使用 StatusBadge 展示，映射关系：
    - `成功` → 成功（绿）
    - `失败` → 错误（红）
  - `失败原因`：状态为失败时显示，最多显示 20 字符，超出省略，鼠标悬停显示完整内容；状态为成功时显示 `-`
  - `下发时间`：格式 `YYYY-MM-DD HH:mm:ss`
  - `操作人`：执行下发操作的用户，自动下发显示`系统`
- **行内操作**：
  - `详情`：点击跳转到配置记录详情页
  - `查看失败原因`：仅状态为失败时显示，点击打开`下发失败详情（弹窗）`

### 3.4 分页

- **分页器**：底部显示，支持切换每页条数（10/20/50/100），默认 20 条/页
- **空状态**：列表为空时显示空状态插画 + 文案「暂无下发记录」

## 4. 配置记录详情页

- **入口**：点击列表页行内`详情`按钮，或通过路由 `/config-records/:id` 访问
- **返回**：左上角返回箭头，返回配置记录列表页

### 4.1 页头区

- **标题**：配置下发详情
- **状态徽章**：成功/失败
- **关键信息**：实例名称、网关名称、下发时间
- **操作按钮**：
  - `重新下发`：仅采集流且状态为失败时显示，主按钮，点击触发重新下发
  - `删除`：次要按钮，点击展开删除确认气泡

### 4.2 基本信息卡片

- **布局**：两列键值对
- **字段**：
  - `实例名称`
  - `设备ID`
  - `网关名称`
  - `下发类型`
  - `下发版本号`
  - `已下发版本号`（下发成功时显示，失败时显示 `-`）
  - `状态`
  - `下发时间`
  - `操作人`
  - `耗时`：单位秒，保留 2 位小数

### 4.3 失败详情卡片

- **显示条件**：仅状态为失败时显示
- **字段**：
  - `失败原因`：完整显示失败原因文本
  - `错误码`：如有，无则显示 `-`
  - `建议操作`：根据失败原因给出的建议，无则显示 `-`

### 4.4 Flow 信息卡片

- **显示条件**：仅采集流类型时显示
- **字段**：
  - `Flow 名称`：格式为「设备实例名称-设备ID」
  - `Flow ID`：Node-RED 中 flow 的唯一标识，无数据显示 `-`
  - `节点数量`：flow 中包含的节点数量，无数据显示 `-`

## 5. 下发配置（单个下发）

> 操作入口在设备实例列表页和设备实例详情页

### 5.1 触发方式

- **设备实例列表页**：行内操作`下发`按钮
- **设备实例详情页**：页头`下发配置`按钮

### 5.2 下发前校验

- 点击下发按钮后，先校验：
  - 实例是否已绑定网关 → 未绑定则提示「请先绑定网关」
  - 网关是否在线 → 离线则提示「网关离线，下发失败」
- 校验通过后，系统自动比对配置版本号：
  - 配置版本号 > 已下发版本号 → 继续下发流程
  - 配置版本号 = 已下发版本号 → 提示「无需下发，配置无变化」，不执行下发

### 5.3 下发中状态

- 按钮文字变为「下发中...」并禁用
- 列表行内显示加载状态

### 5.4 下发结果

- **下发成功**：
  - Toast 提示「下发成功」
  - 刷新列表数据
  - 更新实例的已下发版本号
- **下发失败**：
  - 弹出`下发失败详情（弹窗）`
  - 显示具体失败原因
  - 用户可选择`重试`或`关闭`

---

## 6. 批量下发

> 操作入口在设备实例列表页

### 6.1 触发方式

- 勾选多个设备实例后，点击顶部`批量下发`按钮

### 6.2 下发前校验

- 校验是否至少选中 1 个实例 → 未选中则提示「请先选择要下发的实例」
- 系统自动过滤未绑定网关的实例，弹出提示「已过滤 {N} 个未绑定网关的实例」
- 系统自动过滤网关离线的实例，弹出提示「已过滤 {N} 个网关离线的实例」
- 过滤后无可下发实例时，提示「无可下发的实例」，终止流程

### 6.3 批量下发确认（弹窗）

- **弹窗标题**：确认批量下发
- **弹窗内容**：
  - 提示文字：「即将向 {N} 个实例下发配置，是否继续？」
  - 统计信息：
    - 选中实例数：{N}
    - 可下发实例数：{M}
    - 涉及网关数：{G}
- **底部操作**：
  - `取消`：关闭弹窗
  - `确认下发`：主按钮，点击后开始批量下发

### 6.4 下发过程

- 关闭确认弹窗，显示批量下发进度面板（抽屉或顶部进度条）
- 进度面板展示：
  - 总进度：已完成 / 总数 + 进度条
  - 成功数：{S}
  - 失败数：{F}
- 按网关分组并行下发，同网关内串行下发
- 每条记录单独显示结果

### 6.5 下发完成

- 全部完成后，进度面板显示最终结果：
  - 「下发完成：成功 {S} 个，失败 {F} 个」
  - `查看详情`按钮：点击跳转到配置记录列表页，带上本次下发的筛选条件
  - `关闭`按钮：关闭进度面板
- 刷新设备实例列表数据
- 单个实例失败不影响其他实例的下发

---

## 7. 下发失败详情（弹窗）

- **打开方式**：点击配置记录列表行内`查看失败原因`按钮，或下发失败后自动弹出
- **弹窗宽度**：中等（约 480px）
- **弹窗内容**：
  - 标题：「下发失败」
  - 实例名称：{实例名称}
  - 网关名称：{网关名称}
  - 失败原因：完整显示失败原因文本，红色文字
  - 错误码：如有，无则显示 `-`
  - 建议操作：根据失败原因给出的建议
- **底部操作**：
  - `关闭`：关闭弹窗
  - `重试`：主按钮，仅采集流时显示，点击后重新下发

---

## 8. 删除确认（气泡）

- **触发方式**：点击配置记录详情页`删除`按钮
- **气泡位置**：按钮下方，右对齐
- **关闭方式**：点击取消、点击气泡外部、删除成功后自动关闭
- **气泡内容**：
  - 标题：「确定删除此条记录吗？」
  - 辅助说明：「仅删除平台记录，不影响网关已下发的配置」
  - 操作按钮：
    - `取消`：灰色按钮，关闭气泡
    - `确定`：红色按钮
      - 点击后变为「删除中...」并禁用
      - 调用删除配置记录 API
      - 成功后返回列表页，Toast 提示「删除成功」

---

## 9. 心跳 flow 自动下发（网关激活）

> 此流程无独立页面，在网关新增时自动触发

### 9.1 触发时机

- 用户在「采集网关」页面新增网关，点击`确认`后
- 网关创建成功，系统自动触发心跳 flow 下发

### 9.2 下发结果

- **下发成功**：
  - 网关状态变为「已激活」
  - 记录下发历史（下发类型：心跳流，状态：成功，操作人：系统）
- **下发失败**：
  - 网关创建成功，但状态为「未激活」
  - 记录下发历史（下发类型：心跳流，状态：失败，操作人：系统，记录失败原因）
  - 网关列表中状态显示为「未激活」（灰色徽章）
  - 用户可在网关详情页或配置记录中查看失败原因并重试

---

## 10. 实例删除自动清理 flow

> 此流程无独立页面，在设备实例删除时自动触发

### 10.1 触发时机

- 用户在设备实例页面删除实例，确认删除后

### 10.2 清理逻辑

- 系统调用 Node-RED API 删除对应 flow
- Flow 命名规则：`设备实例名称-设备ID`
- **flow 存在**：删除成功，继续删除平台实例记录
- **flow 不存在**：跳过删除 flow，继续删除平台实例记录，记录日志「网关配置不存在，跳过删除 flow」
- **网关离线**：仍删除平台实例记录，记录日志「网关离线，未删除对应 flow」

### 10.3 用户反馈

- 删除成功 Toast 提示「删除成功」
- 网关离线或 flow 不存在时，Toast 仍提示「删除成功」，详细原因可在操作日志中查看
  - `待确认`：是否需要在 Toast 中提示部分失败信息

## 11. 业务规则与状态反馈

### 11.1 配置版本规则

- 每次实例配置变化产生新配置版本号，版本号递增
- 下发成功后，更新该实例的已下发版本号为当前配置版本号
- 下发失败时，已下发版本号保持不变
- 下发前比对：配置版本号 > 已下发版本号才执行下发，相等则提示「无需下发」

### 11.2 Flow 命名规则

- 每个实例对应一个独立 flow
- Flow 名称固定为「设备实例名称-设备ID」
- 重新下发时全量覆盖对应实例 flow，以平台配置为准

### 11.3 批量下发规则

- 按网关分组并行下发，不同网关互不阻塞
- 同一网关内串行下发
- 单个实例失败不影响同网关内后续实例的下发
- 每条记录单独记录下发历史

### 11.4 通用反馈

- **操作成功**：Toast 提示成功信息，3 秒后消失
- **操作失败**：Toast 或弹窗显示具体错误原因
- **加载状态**：按钮置灰 + 文字变化（如「下发中...」）
- **空状态**：列表为空时显示空状态插画 + 文案
- **搜索无结果**：显示「未找到匹配的记录」
