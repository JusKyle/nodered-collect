# 实时状态推送 — 功能需求说明书

## 1. 背景与价值

**为什么做**：实时状态推送采用 SSE（Server-Sent Events）技术，将后端状态变化实时推送到前端，替代传统的轮询刷新机制。用户无需手动刷新页面，始终看到最新的网关状态、采集数据和下发结果。

**做成什么样**：前端页面打开时自动建立 SSE 连接，断开后自动重试，用户零操作即可获得实时更新体验。

**关联**：被边缘网关管理、设备实例管理、配置下发与同步模块依赖。

---

## 2. 范围

### 本次做

- 五类 SSE 事件（网关状态/性能、采集值、下发状态、缓存状态）的实时推送
- 前端 SSE 自动连接、断开检测、自动重试
- 有限重试策略（指数退避，最多 10 次）
- 每个标签页独立 SSE 连接
- 采集数据按固定频率（1秒/次）批量聚合后推送
- 页面关闭自动断开连接

### 本次不做

- 用户之间的事件隔离（后续权限系统）
- 历史推送记录查询
- WebSocket 或其他推送协议
- 多标签页共享连接方案

## 4. 需求详述

### 4.1 核心流程

> 这里只描述**主线路径的全局视角**——用户从哪里开始、经过哪些关键节点、到哪里结束。

**SSE 连接建立流程**
1. 用户打开平台页面
2. 前端自动建立 SSE 连接
3. 每个标签页独立建立连接
4. 后端建立推送通道

**事件推送流程**
1. 后端检测到事件发生
2. 根据事件类型推送到相关客户端
3. 前端接收并更新页面

**重连流程**
1. SSE 连接断开
2. 前端按指数退避重试（1s → 2s → 4s...最长 30s）
3. 重试成功则继续
4. 10 次失败后停止，显示刷新提示

### 4.2 交互规则

- 连接断开时前端显示重连状态
- 10 次重试失败后显示"连接断开，请刷新页面"
- 页面关闭时自动断开连接
- 多标签页各独立接收推送

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 每个浏览器标签页独立建立 SSE 连接 |
| BR-002 | 重试策略：指数退避（最长 30 秒间隔），最多重试 10 次 |
| BR-003 | 采集值事件按 1 秒频率批量聚合推送，非逐条推送 |
| BR-004 | 五类事件（gateway_status / gateway_performance / device_point_values / deploy_status / cache_status）独立推送 |

## 5. 验收标准

> AC 是这份文档的核心交付物。后续的技术方案、任务拆分、TDD 测试用例全部从这里派生。

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 自动建立 SSE 连接
  - **Given** 用户打开平台页面
  - **When** 页面加载完成
  - **Then** 前端自动建立 SSE 连接

- [ ] **AC-002**: 网关状态事件推送
  - **Given** SSE 连接已建立
  - **When** 网关上报心跳
  - **Then** 后端立即推送"网关状态事件"到所有标签页

- [ ] **AC-003**: 状态变化实时更新
  - **Given** 网关状态变化（在线↔离线）
  - **When** 用户查看任一标签页的网关列表
  - **Then** 列表实时更新状态，无需手动刷新

- [ ] **AC-004**: 网关性能事件推送
  - **Given** SSE 连接已建立
  - **When** 网关上报心跳（含性能数据）
  - **Then** 后端立即推送"网关性能事件"到所有标签页

- [ ] **AC-005**: 性能数据实时更新
  - **Given** 网关性能数据到达
  - **When** 用户打开网关详情页
  - **Then** 详情页实时更新 CPU、内存、磁盘使用率

- [ ] **AC-006**: 采集值事件聚合推送
  - **Given** 网关上报采集数据
  - **When** 后端接收并写入时序库
  - **Then** 后端按 1 秒频率批量聚合后推送"采集值事件"到所有标签页

- [ ] **AC-007**: 采集值实时展示
  - **Given** 采集值事件到达
  - **When** 用户查看实例详情页的点位列表
  - **Then** 各点位实时显示最新值和更新时间

- [ ] **AC-008**: 下发成功推送
  - **Given** 下发配置执行完成
  - **When** 下发成功
  - **Then** 后端推送"下发状态事件"，相关页面显示"下发成功"

- [ ] **AC-009**: 下发失败推送
  - **Given** 下发配置执行完成
  - **When** 下发失败
  - **Then** 后端推送"下发状态事件"（含失败原因），相关页面显示"下发失败"和原因

- [ ] **AC-010**: 缓存状态事件推送
  - **Given** 断网缓存数据变化
  - **When** 网关缓存条数 / 存储空间 / 补发进度发生变化
  - **Then** 后端推送"缓存状态事件"，网关详情页实时更新缓存信息

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-011**: 临时断网自动重连
  - **Given** SSE 连接因网络抖动断开
  - **When** 断开后 1 秒内恢复
  - **Then** 前端自动重连成功，继续接收推送

- [ ] **AC-012**: 彻底断开提示刷新
  - **Given** SSE 连接持续断开
  - **When** 前端按指数退避重试（1s → 2s → 4s...最长 30s）
  - **Then** 连续 10 次重试失败后停止重试，显示"连接断开，请刷新页面"

- [ ] **AC-013**: 多标签页独立接收
  - **Given** 用户打开多个浏览器标签页
  - **When** 后端推送事件
  - **Then** 每个标签页独立接收该事件

- [ ] **AC-014**: 页面关闭断开连接
  - **Given** 用户离开页面
  - **When** 页面关闭或切换到其他站点
  - **Then** 当前标签页的 SSE 连接自动断开

- [ ] **AC-015**: 高频采集聚合推送
  - **Given** 高频采集场景（点位采集间隔 < 1秒）
  - **When** 后端接收到多条采集数据
  - **Then** 按 1 秒频率批量聚合后再推送，不逐条推送

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-016**: 网关状态推送频率 ← BR-001
  - **Given** SSE 推送机制
  - **Then** 网关状态 / 性能事件随每次心跳（30秒/次）触发，同次推送

- [ ] **AC-017**: 采集值聚合推送 ← BR-003
  - **Given** SSE 推送机制
  - **Then** 采集值事件按固定频率（1秒/次）批量推送，不逐条实时推送

- [ ] **AC-018**: 下发状态立即推送 ← BR-004
  - **Given** SSE 推送机制
  - **Then** 下发状态事件在下发完成时立即推送，不延迟

- [ ] **AC-019**: 缓存状态立即推送 ← BR-004
  - **Given** SSE 推送机制
  - **Then** 缓存状态事件在缓存数据变化时立即推送

- [ ] **AC-020**: 重试策略 ← BR-002
  - **Given** SSE 连接管理
  - **Then** 重试策略采用指数退避（最长 30 秒间隔），最多重试 10 次

- [ ] **AC-021**: 独立连接 ← BR-001
  - **Given** SSE 连接管理
  - **Then** 每个浏览器标签页独立建立 SSE 连接，不共享

- [ ] **AC-022**: 事件类型区分 ← BR-004
  - **Given** 事件类型
  - **Then** 五类事件（gateway_status / gateway_performance / device_point_values / deploy_status / cache_status）独立推送，前端按类型区分处理
