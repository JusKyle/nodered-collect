# 实时状态推送 · 需求文档

## 1. 功能概述

实时状态推送模块采用 Server-Sent Events（SSE）技术，将后端的状态变化实时推送到前端，替代传统的轮询刷新机制。推送内容涵盖网关状态与性能、设备采集值、下发配置状态、断网缓存状态五类事件。前端页面打开时自动建立 SSE 连接，断开后自动重试，确保用户始终看到最新的系统状态。

## 2. 场景与用户

### 2.1 目标用户

- 工业自动化工程师 — 关注采集数据实时值和设备运行状态
- IoT 系统运维人员 — 关注网关在线状态、性能指标和下发结果
- 数据采集系统管理员 — 关注整体系统运行状态和缓存情况

### 2.2 典型场景

**场景一：网关状态实时监控**
运维人员打开网关列表页面，SSE 连接自动建立。网关每次上报心跳（30秒/次），列表实时更新在线/离线状态、最后心跳时间，无需手动刷新。

**场景二：性能指标实时查看**
运维人员打开网关详情页，每次心跳到达时，CPU、内存、磁盘使用率等性能指标实时更新趋势图。

**场景三：采集数据实时展示**
工程师打开某设备实例详情页，网关采集到点位数据后，点位列表中的实时值和更新时间自动刷新，无需轮询。

**场景四：下发状态实时反馈**
管理员点击下发配置，系统执行下发后立即推送结果，页面实时显示"下发成功"或"下发失败及原因"。

**场景五：断网缓存状态监控**
运维人员关注某网关的断网缓存状态，缓存条数增加或补发进行时，页面实时更新缓存状态信息。

**场景六：临时断网自动恢复**
网络临时断开导致 SSE 连接中断，前端按指数退避策略自动重试，网络恢复后连接自动重建，推送恢复正常。

## 3. 核心流程

### 3.1 SSE 连接管理流程

1. 用户打开平台页面（网关列表、实例详情等），前端自动建立 SSE 连接
2. 每个浏览器标签页独立建立连接，不共享
3. 后端接收连接请求，为该标签页建立长期推送通道
4. 页面关闭或用户离开时，前端主动断开 SSE 连接

### 3.2 连接断开重试流程

1. SSE 连接因网络抖动或服务器问题断开
2. 前端检测到断开事件，启动重试策略
3. 按指数退避间隔重试：1秒 → 2秒 → 4秒 → 8秒 → ... 最长 30 秒
4. 重试成功 → 推送恢复，无数据丢失
5. 连续 10 次重试失败 → 停止重试，页面显示"连接断开，请刷新页面"

### 3.3 事件类型与推送逻辑

**网关状态事件**
- 触发器：网关每次上报心跳（30秒/次）
- 内容：网关ID、状态（在线/离线）、变化时间
- 推送时机：每次心跳上报后立即推送

**网关性能事件**
- 触发器：网关每次上报心跳（30秒/次）
- 内容：网关ID、CPU使用率、内存使用率、磁盘使用率、磁盘剩余空间
- 推送时机：每次心跳上报后立即推送（与网关状态事件同次触发）

**采集值事件**
- 触发器：网关上报采集数据，后端写入时序库
- 内容：设备实例ID、点位列表（点位ID、值、时间戳）
- 推送时机：后端按固定频率（1秒/次）批量聚合后再推送，不逐条实时推送

**下发状态事件**
- 触发器：下发配置执行完成
- 内容：实例ID、下发状态（成功/失败）、失败原因
- 推送时机：下发完成时立即推送，不延迟

**缓存状态事件**
- 触发器：网关缓存数据发生变化（新增/补发/清理）
- 内容：网关ID、缓存条数、存储空间、是否补发中、补发进度
- 推送时机：缓存变化时立即推送

## 4. 事件数据结构

### 4.1 网关状态事件

| 字段 | 类型 | 说明 |
|------|------|------|
| eventType | 枚举 | gateway_status |
| gatewayId | 字符串 | 网关ID |
| status | 枚举 | online / offline |
| timestamp | 时间戳 | 状态变化时间 |

### 4.2 网关性能事件

| 字段 | 类型 | 说明 |
|------|------|------|
| eventType | 枚举 | gateway_performance |
| gatewayId | 字符串 | 网关ID |
| cpuUsage | 浮点数 | CPU使用率（%） |
| memoryUsage | 浮点数 | 内存使用率（%） |
| diskUsage | 浮点数 | 磁盘使用率（%） |
| diskFree | 浮点数 | 磁盘剩余空间 |
| timestamp | 时间戳 | 上报时间 |

### 4.3 采集值事件

| 字段 | 类型 | 说明 |
|------|------|------|
| eventType | 枚举 | device_point_values |
| deviceInstanceId | 字符串 | 设备实例ID |
| points | 数组 | 点位列表 |
| ├ pointId | 字符串 | 点位ID |
| ├ value | 任意 | 点位值 |
| └ timestamp | 时间戳 | 采集时间 |

### 4.4 下发状态事件

| 字段 | 类型 | 说明 |
|------|------|------|
| eventType | 枚举 | deploy_status |
| instanceId | 字符串 | 实例ID |
| status | 枚举 | success / failed |
| reason | 字符串 | 失败原因（成功时为空） |
| timestamp | 时间戳 | 完成时间 |

### 4.5 缓存状态事件

| 字段 | 类型 | 说明 |
|------|------|------|
| eventType | 枚举 | cache_status |
| gatewayId | 字符串 | 网关ID |
| cachedCount | 整数 | 当前缓存条数 |
| storageUsed | 浮点数 | 已用存储空间 |
| isReplaying | 布尔 | 是否正在补发 |
| replayProgress | 浮点数 | 补发进度（%） |
| timestamp | 时间戳 | 缓存变化时间 |

## 5. 边界与异常处理

### 5.1 连接管理

| 异常场景 | 处理方式 |
|---------|---------|
| SSE 连接因网络抖动断开 | 前端自动重试，指数退避（1s→2s→4s...最长30s间隔） |
| 连续 10 次重试失败 | 停止重试，显示"连接断开，请刷新页面" |
| 页面关闭或用户离开 | 前端自动断开 SSE 连接 |
| 用户打开多个标签页 | 每个标签页独立建立 SSE 连接，各自接收推送 |

### 5.2 事件推送

| 异常场景 | 处理方式 |
|---------|---------|
| 短时间大量采集数据到达 | 后端按 1 秒频率批量聚合后推送，不逐条推送 |
| 网关离线停发心跳 | 无网关状态事件推送，前端保持当前离线状态 |
| 网络恢复后存在缓存数据 | 后端先推送缓存状态事件，再推送补发后的采集值事件 |

## 6. 业务规则

### 6.1 连接规则

- 每个浏览器标签页独立建立 SSE 连接
- 页面打开自动连接，页面关闭自动断开
- 重试策略：指数退避（最长 30 秒间隔），最多重试 10 次

### 6.2 推送规则

- 网关状态和性能事件：随心跳（30秒/次）触发，同次处理
- 采集值事件：后端按 1 秒频率批量聚合后推送，非逐条推送
- 下发状态事件：完成时立即推送
- 缓存状态事件：变化时立即推送

### 6.3 事件类型

- 五类独立事件：gateway_status、gateway_performance、device_point_values、deploy_status、cache_status
- 前端按事件类型区分处理

## 7. 范围界定

### 7.1 本次做

- 五类 SSE 事件（网关状态/性能、采集值、下发状态、缓存状态）的实时推送
- 前端 SSE 自动连接、断开检测、自动重试
- 有限重试策略（指数退避，最多 10 次）
- 每个标签页独立 SSE 连接
- 采集数据按固定频率（1秒/次）批量聚合后推送
- 页面关闭自动断开连接

### 7.2 本次不做

- 用户之间的事件隔离（后续权限系统）
- 历史推送记录查询
- WebSocket 或其他推送协议
- 多标签页共享连接方案

## 8. 验收标准（AC）

### 8.1 Happy Path

| 编号 | 验收标准 |
|------|---------|
| AC-001 | Given 用户打开平台页面，When 页面加载完成，Then 前端自动建立 SSE 连接。 |
| AC-002 | Given SSE 连接已建立，When 网关上报心跳，Then 后端立即推送"网关状态事件"到所有标签页。 |
| AC-003 | Given 网关状态变化（在线↔离线），When 用户查看任一标签页的网关列表，Then 列表实时更新状态，无需手动刷新。 |
| AC-004 | Given SSE 连接已建立，When 网关上报心跳（含性能数据），Then 后端立即推送"网关性能事件"到所有标签页。 |
| AC-005 | Given 网关性能数据到达，When 用户打开网关详情页，Then 详情页实时更新 CPU、内存、磁盘使用率。 |
| AC-006 | Given 网关上报采集数据，When 后端接收并写入时序库，Then 后端按 1 秒频率批量聚合后推送"采集值事件"到所有标签页。 |
| AC-007 | Given 采集值事件到达，When 用户查看实例详情页的点位列表，Then 各点位实时显示最新值和更新时间。 |
| AC-008 | Given 下发配置执行完成，When 下发成功，Then 后端推送"下发状态事件"，相关页面显示"下发成功"。 |
| AC-009 | Given 下发配置执行完成，When 下发失败，Then 后端推送"下发状态事件"（含失败原因），相关页面显示"下发失败"和原因。 |
| AC-010 | Given 断网缓存数据变化，When 网关缓存条数/存储空间/补发进度发生变化，Then 后端推送"缓存状态事件"，网关详情页实时更新缓存信息。 |

### 8.2 Edge & Error Cases

| 编号 | 验收标准 |
|------|---------|
| AC-011 | Given SSE 连接因网络抖动断开，When 断开后 1 秒内恢复，Then 前端自动重连成功，继续接收推送。 |
| AC-012 | Given SSE 连接持续断开，When 前端按指数退避重试（1s→2s→4s...最长30s），Then 连续 10 次重试失败后停止重试，显示"连接断开，请刷新页面"。 |
| AC-013 | Given 用户打开多个浏览器标签页，When 后端推送事件，Then 每个标签页独立接收该事件。 |
| AC-014 | Given 用户离开页面，When 页面关闭或切换到其他站点，Then 当前标签页的 SSE 连接自动断开。 |
| AC-015 | Given 高频采集场景（点位采集间隔 < 1秒），When 后端接收到多条采集数据，Then 按 1 秒频率批量聚合后再推送，不逐条推送。 |

### 8.3 Business Rules

| 编号 | 验收标准 |
|------|---------|
| AC-016 | Given SSE 推送机制，Then 网关状态/性能事件随每次心跳（30秒/次）触发，同次推送。 |
| AC-017 | Given SSE 推送机制，Then 采集值事件按固定频率（1秒/次）批量推送，不逐条实时推送。 |
| AC-018 | Given SSE 推送机制，Then 下发状态事件在下发完成时立即推送，不延迟。 |
| AC-019 | Given SSE 推送机制，Then 缓存状态事件在缓存数据变化时立即推送。 |
| AC-020 | Given SSE 连接管理，Then 重试策略采用指数退避（最长 30 秒间隔），最多重试 10 次。 |
| AC-021 | Given SSE 连接管理，Then 每个浏览器标签页独立建立 SSE 连接，不共享。 |
| AC-022 | Given 事件类型，Then 五类事件（gateway_status / gateway_performance / device_point_values / deploy_status / cache_status）独立推送，前端按类型区分处理。 |
