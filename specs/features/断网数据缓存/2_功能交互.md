# 断网数据缓存 - 页面功能交互说明

## 1. 页面综述

- **功能描述**：管理边缘网关断网数据缓存策略，包括两级缓存开关配置、缓存状态实时监控、补发进度追踪和历史补发记录查看，支持手动清空缓存。
- **页面类型**：后台管理类 (Admin / Table-First) + 网关详情页卡片
- **目标用户**：IoT 系统运维人员、数据采集系统管理员
- **核心目标**：用户可以在平台配置缓存策略，实时监控缓存状态和补发进度，确保断网期间数据不丢失。
- **本阶段范围**：两级缓存开关配置（平台级 + 网关级）、缓存状态展示、补发进度展示、历史补发记录、清空缓存

---

## 2. 页面结构

- **页面标题**：网关配置
- **页面副标题**：配置平台级缓存默认策略
- **主体布局**：导航栏 + 顶部筛选区 + 配置表单区
- **主要页面/区域**：
  - 导航栏：采集网关，注册码，设备模型，设备实例，配置记录，系统配置
  - 系统配置 → 网关配置页（平台级配置）
  - 网关详情页 → 缓存配置卡片（网关级配置）
  - 网关详情页 → 缓存状态卡片（状态展示）
  - 清空缓存（弹窗）

---

## 3. 系统配置 - 网关配置页

> 对应需求：AC-001、AC-022、AC-027

### 3.1 页面入口

- **导航入口**：顶部导航栏 → `系统配置` → 左侧菜单 `网关配置`
- **页面定位**：平台级默认配置，作为所有网关的缓存策略默认值

### 3.2 缓存配置区

- **标题**：断网缓存配置
- **配置说明**：
  - 此配置为平台级默认值，所有网关默认使用此配置
  - 网关可单独配置覆盖此默认值（在网关详情页配置）
  - 配置优先级：网关级 > 平台级
- **表单字段**：
  - `缓存开关`：Switch 开关，控制平台级缓存默认启用/禁用
    - 默认：关闭
    - 开关切换后立即生效，无需保存按钮
    - 切换提示：
      - 开启时：「平台级缓存已开启，新网关将默认启用缓存」
      - 关闭时：「平台级缓存已关闭，新网关将默认禁用缓存」
  - `保存期限`：下拉选，当缓存开关开启时显示
    - 选项：`7 天`、`15 天`、`30 天`、`60 天`、`90 天`
    - 默认：`15 天`
    - 说明：超过保留期限的缓存数据自动清理
  - `补发速率`：数字输入框，当缓存开关开启时显示
    - 单位：条/秒
    - 范围：1-500
    - 默认：100
    - 说明：网络恢复后按此速率补发缓存数据
- **保存操作**：
  - `保存`：主按钮（缓存开关开启时显示）
    - 点击后按钮变为「保存中...」并禁用
    - 调用保存平台级配置 API
    - 成功后 Toast 提示「保存成功」
    - 配置立即生效，无需重启网关

## 4. 网关详情页 - 缓存配置卡片

> 对应需求：AC-002、AC-003、AC-027、AC-034、AC-035
> 页面入口在边缘网关管理模块，此处仅描述缓存相关交互

### 4.1 卡片标题

- **标题**：缓存配置
- **右上角状态**：显示当前生效的配置来源
  - `网关级`：使用网关级配置（蓝色标签）
  - `平台级`：使用平台级默认配置（灰色标签）

### 4.2 配置表单

- **配置说明**：
  - 网关级配置可覆盖平台级默认值
  - 配置优先级：网关级 > 平台级
  - 配置立即生效，无需重新下发配置
- **表单字段**：
  - `缓存开关`：Switch 开关，控制网关级缓存启用/禁用
    - 默认：使用平台级配置
    - 三态显示：
      - 开启：网关级开启
      - 关闭：网关级关闭
      - 继承：使用平台级配置（灰色状态）
    - 切换后立即生效，无需保存按钮
  - `保存期限`：下拉选，当缓存开关开启时显示
    - 选项：`7 天`、`15 天`、`30 天`、`60 天`、`90 天`
    - 默认：继承平台级配置（显示平台级值）
  - `补发速率`：数字输入框，当缓存开关开启时显示
    - 单位：条/秒
    - 范围：1-500
    - 默认：继承平台级配置（显示平台级值）
- **保存操作**：
  - `保存`：主按钮（有配置变更时显示）
    - 点击后按钮变为「保存中...」并禁用
    - 调用保存网关级配置 API
    - 成功后 Toast 提示「保存成功」
    - 配置立即生效，无需重新下发配置

## 5. 网关详情页 - 缓存状态卡片

> 对应需求：AC-012、AC-013、AC-014、AC-015、AC-016、AC-025

### 5.1 卡片标题

- **标题**：缓存状态
- **右上角状态**：显示缓存开关状态
  - `已开启`：缓存功能正常运行（绿色标签）
  - `已关闭`：缓存功能已禁用（灰色标签）
  - `缓存中`：当前正在缓存数据（橙色标签，断网时显示）
  - `补发中`：当前正在补发数据（蓝色标签，补发时显示）

### 5.2 当前生效配置

- **布局**：两列键值对
- **字段**：
  - `缓存开关`：当前生效的开关状态（开启/关闭）+ 来源标注（网关级/平台级）
  - `保存期限`：当前生效的保存期限 + 来源标注
  - `补发速率`：当前生效的补发速率 + 来源标注
- **来源标注规则**：
  - 网关级配置：显示「网关级」（蓝色）
  - 平台级配置：显示「平台级」（灰色）

### 5.3 缓存统计

- **统计项**：
  - `缓存条数`：当前本地缓存的数据条数，实时更新（SSE推送）
  - `存储占用`：已用空间 / 总上限
    - 带进度条显示
    - 百分比显示（如 45%）
    - 超过 80% 时进度条变橙色警告
    - 超过 95% 时进度条变红色警告
  - `最早数据时间`：缓存中最早一条数据的采集时间
  - `最新数据时间`：缓存中最新一条数据的采集时间
- **空状态**：
  - 缓存条数为 0 时显示空状态插画 + 文案「暂无缓存数据」

### 5.4 补发进度

- **显示条件**：网关正在补发缓存数据时显示
- **进度条**：
  - 格式：已补发 / 总条数 + 百分比（如 350 / 1000 (35%)）
  - 进度条实时更新（SSE推送）
  - 补发按采集时间先后顺序（最早的数据先补发）
  - 按配置速率（1-500条/秒）补发
- **补发状态**：
  - `补发中`：蓝色标签，进度条动态增长
  - `已完成`：绿色标签，进度条满
  - `已暂停`：灰色标签（补发中断时显示）
- **补发速率显示**：当前补发速率（如 100 条/秒）

### 5.5 清空缓存

- **按钮**：`清空缓存`（红色警告按钮）
- **触发条件**：缓存条数 > 0 时显示
- **网关离线处理**：
  - 网关离线时按钮仍可用，但显示提示「网关离线，指令将在上线后执行」
  - 点击后清空指令缓存到网关，网关上线后执行
- **二次确认弹窗**：点击按钮后弹出

### 5.6 历史补发记录

- **标题**：历史补发记录，右上角显示总记录数
- **列表展示**：
  - 排序规则：按 `补发时间` 倒序排列
  - 列表字段：
    - `补发时间`：格式 `YYYY-MM-DD HH:mm:ss`
    - `补发条数`：本次补发的数据条数
    - `补发耗时`：补发用时（秒）
    - `补发状态`：使用标签展示
      - `成功` → 绿色标签
      - `部分成功` → 橙色标签（补发中断）
      - `失败` → 红色标签
    - `失败原因`：失败或部分成功时显示，最多 20 字符，超出省略
- **分页**：支持分页，每页 10 条
- **空状态**：无历史记录时显示空状态插画 + 文案「暂无补发记录」

## 6. 清空缓存（弹窗）

> 对应需求：AC-016、AC-017、AC-026

### 6.1 弹窗触发

- **触发方式**：点击缓存状态卡片的 `清空缓存` 按钮
- **弹窗宽度**：中等（约 400px）

### 6.2 弹窗内容

- **标题**：清空缓存确认
- **警告提示**：
  - 红色警告图标 + 文案：「清空缓存将删除所有本地缓存数据，网络恢复后这些数据将无法补发」
- **统计信息**：
  - 当前缓存条数：{N} 条
  - 最早数据时间：{时间}
  - 最新数据时间：{时间}
- **网关离线提示**：
  - 网关离线时显示：「网关离线，清空指令将在网关上线后执行」
- **操作按钮**：
  - `取消`：灰色按钮，关闭弹窗，缓存数据保留
  - `确认清空`：红色按钮
    - 点击后按钮变为「清空中...」并禁用
    - 调用清空缓存 API
    - 成功后 Toast 提示「清空指令已发送」，关闭弹窗
    - 网关离线时提示「清空指令已缓存，网关上线后执行」

## 8. 业务规则与状态反馈

### 8.1 缓存开关优先级规则

- **两级开关**：网关级 > 平台级
- **默认值**：两级默认均为关闭
- **判定逻辑**：
  1. 如果网关级开关有配置值（非继承），以网关级配置为准
  2. 否则，使用平台级默认值
- **配置来源标注**：
  - 网关级配置：显示「网关级」标签
  - 平台级配置：显示「平台级」标签
- **配置生效时机**：保存后立即生效，无需重新下发配置

### 8.2 断网检测与缓存触发规则

- **断网判定条件**：
  - MQTT 连接断开
  - 或 心跳发送失败（N 次重试后）
- **缓存触发**：
  - 断网判定后，检查缓存开关（网关级 > 平台级）
  - 开关开启时，将数据写入本地 SQLite
- **缓存字段**：设备实例 ID、点位 ID、点位值、采集时间戳、数据质量

### 8.3 缓存补发规则

- **补发触发**：MQTT 重连成功后自动立即补发
- **补发顺序**：严格按采集时间先后顺序（最早先补发）
- **补发速率**：1-500 条/秒，使用配置速率
- **补发 Topic**：使用独立 Topic `device/{deviceId}/replay`
- **消息标记**：每条补发消息含 `replay: true` 标记
- **补发确认**：收到 MQTT PUBACK 后删除本地缓存记录
- **去重规则**：按点位 ID + 采集时间戳去重

### 8.4 存储上限规则

- **过期清理**：超过保留期限的缓存数据自动清理
- **上限处理**：
  - 达到物理存储上限时，停止写入新数据
  - 或覆盖最旧数据（取决于配置）
- **上限警告**：
  - 存储占用 > 80%：橙色警告
  - 存储占用 > 95%：红色警告

### 8.5 清空缓存规则

- **二次确认**：清空缓存需二次确认弹窗
- **网关离线**：清空指令缓存到网关，上线后执行
- **清空后行为**：清空不影响后续缓存行为，断网期间新数据继续缓存

### 8.6 通用反馈

- **操作成功**：Toast 提示成功信息，3 秒后消失
- **操作失败**：Toast 或弹窗显示具体错误原因
- **加载状态**：按钮置灰 + 文字变化（如「保存中...」「清空中...」）
- **配置生效提示**：配置保存成功后提示「保存成功，配置已立即生效」
