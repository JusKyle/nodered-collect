# 边缘网关管理 - 页面功能交互说明

## 1. 页面综述

- **功能描述**：统一管理分布式部署的 Node-RED 边缘网关，支持网关注册接入、实时状态监控、性能趋势查看和断网缓存控制。
- **页面类型**：后台管理类 (Admin / Table-First)
- **目标用户**：工业自动化工程师、IoT 系统运维人员、数据采集系统管理员
- **核心目标**：用户可以在平台上完成网关的注册接入、实时状态监控、性能趋势查看，无需逐台登录网关操作。

---

## 2. 页面结构

- **页面标题**：采集网关
- **页面副标题**：管理所有采集网关的注册和状态
- **主体布局**：导航栏 + 顶部操作区 + 数据表格区 
- **主要页面/区域**：
  - 导航栏：采集网关，注册码，设备模型，设备实例，配置记录，系统配置
  - 网关列表页（主页面）
  - 激活网关（弹窗）
  - 测试连接结果（行内提示）
  - 删除确认（气泡）
  - 网关详情页（二级页面）
---

## 3. 网关列表页

### 3.1 搜索/筛选区

- **搜索字段**：
  - `网关名称`：输入框，默认空，模糊搜索
  - `IP地址`：输入框，默认空，模糊搜索
- **筛选字段**：
  - `NR版本`：下拉选，默认`全部`，选项：`v3.1.2`、`v3.1.1`
  - `状态`：下拉选，默认`全部`，选项：`全部`、`在线`、`离线`
- **交互**：
  - 输入搜索关键词后需点击查询按钮
  - 选择筛选条件后需点击查询按钮
  - 搜索与筛选条件叠加生效

### 3.2 操作按钮区

- **`新增网关`**：主按钮（primary），点击打开`新增网关（弹窗）`

### 3.3 列表展示区

- **排序规则**：默认按 `网关名称` 顺序排列。
- **列表字段**：
  - `名称`：网关名称，加粗显示，可点击跳转到网关详情页
  - `ip地址`：Node-RED IP 地址或域名,无数据显示 `-`
  - `状态`：使用 StatusBadge 展示，映射关系：
    - `ONLINE` → 在线（绿）
    - `OFFLINE` → 离线（灰）
  - `NR 版本`：Node-RED 版本号，无数据显示 `-`
  - `流数`：当前 flow 数量，无数据显示 `-`
  - `最后心跳`：最后一次心跳时间，格式 `YYYY-MM-DD HH:mm`，无数据显示 `-`
- **行内操作**：
  - `测试`：点击发起连接测试，行内显示结果
  - `设备`：点击跳转到设备实例列表页，带上当前网关筛选条件
  - `删除`：点击展开删除确认气泡

### 3.4 实时更新机制

- **SSE 订阅**：页面挂载时建立 SSE 连接，订阅网关状态变化事件
- **更新触发**：
  - 收到心跳时更新最后心跳时间、状态、NR 版本、IP、流数
  - 状态变化时（在线↔离线）实时更新 StatusBadge
- **断线重连**：SSE 连接断开时自动重试（指数退避）

## 4. 新增网关（弹窗）

- **打开方式**：点击列表页 `新增网关` 按钮
- **弹窗宽度**：中等（约 448px）
- **表单字段**：
  - `网关名称`：必填，输入框，最多 50 字符
  - `ip地址`：必填，输入框，合法 IP 或域名格式校验
  - `端口`：必填，数字输入框，默认 1880，范围 1-65535
- **底部操作**：
  - `取消`：关闭弹窗，重置表单，清除测试结果
  - `确认`：
    - 按钮变为「激活中...」并禁用
    - 平台自动生成 Token
    - 调用激活网关 API（推送配置到 Node-RED）
    - 激活成功后刷新列表，关闭弹窗
- **校验规则**：
  - 实时校验（输入时）+ 提交时全量校验
  - 错误提示：输入框红框 + 下方红色错误文字

## 5. 测试连接

### 5.1 测试项说明

测试连接包含以下三项，分项展示结果：

| 测试项 | 说明 | 成功条件 |
|--------|------|----------|
| `Node-RED 可达性` | 测试平台能否通过 HTTP 访问 Node-RED | 地址可访问、API 调用成功 |
| `MQTT 连接` | 测试网关侧插件能否正常连接 MQTT Broker | MQTT 连接成功 |
| `插件状态` | 测试 nodered-plugin 插件是否正常运行 | 插件已安装且在运行中 |

### 5.2 行内测试（列表页）

- **触发方式**：点击列表行内 `测试连接` 按钮
- **测试中状态**：
  - 按钮文字变为「测试中...」
  - 按钮禁用
- **结果展示**：
  - 操作列下方展开测试结果面板
  - 三项测试分项显示，每项包含：状态图标 + 测试项名称 + 结果说明
  - 每项状态：
    - 成功：绿色勾选图标 + 绿色文字
    - 失败：红色叉号图标 + 红色错误信息
    - 测试中：加载动画 + 「测试中...」
  - 全部成功时底部显示绿色总结：「全部测试通过」
  - 有失败项时底部显示红色总结：「{N} 项测试未通过」

---

## 6. 删除确认（气泡）

- **触发方式**：点击列表行内删除图标按钮
- **气泡位置**：按钮下方，右对齐
- **关闭方式**：点击取消、点击气泡外部、删除成功后自动关闭
- **气泡内容**：
  - 标题：「确定删除网关「{网关名称}」吗？」
  - 辅助说明：「已关联的设备实例将保留不变」
  - 操作按钮：
    - `取消`：灰色按钮，关闭气泡
    - `确定`：红色按钮
      - 点击后变为「删除中...」并禁用
      - 调用删除网关 API
      - 成功后刷新列表，关闭气泡

---

## 7. 网关详情页

- **入口**：点击列表页网关名称，或通过路由 `/gateways/:id` 访问
- **返回**：左上角返回箭头，返回网关列表页

### 7.1 页头区

- **网关名称**：大标题
- **状态徽章**：在线/离线
- **关键信息**：Node-RED 版本、IP 地址、流数
- **操作按钮**：`测试连接`、`已下发设备`、`删除`

### 7.2 基本信息卡片

- **布局**：两列键值对
- **字段**：
  - `网关名称`
  - `地址`
  - `端口`
  - `NR版本`
  - `流数`
  - `最后心跳`
  - `创建时间`

### 7.3 性能趋势卡片

> 对应需求：AC-021、AC-022、AC-023

- **时间范围选择器**：
  - 选项：`5 分钟`、`30 分钟`、`1 小时`、`6 小时`、`24 小时`、`7 天`
  - 默认：`5 分钟`
- **指标切换 Tab**：
  - `CPU 使用率`（%）
  - `内存使用率`（%）
  - `磁盘使用率`（%）
  - `网络IO`（字节/s）
  - 支持多选叠加对比
- **图表区域**：
  - 类型：面积图（AreaChart）
  - X 轴：时间
  - Y 轴：百分比（0-100%）
  - Hover 显示具体数值
- **数据说明**：
  - 性能数据随心跳上报（每 30 秒一次）
  - 性能历史数据最长保留 7 天，超过自动清理

### 7.4 缓存配置卡片

> 对应需求：AC-007a、AC-008、AC-009、AC-016

- **网关缓存配置**：配置网关级断网缓存参数
- **表单字段**：
  - `缓存开关`：Switch 开关，控制网关级缓存启用/禁用
    - 开关切换后立即生效，无需重新下发配置
  - `保存期限`：下拉选，当缓存开关开启时显示
    - 选项：`7 天`、`15 天`、`30 天`、`60 天`、`90 天`
    - 默认：`15 天`
  - `补发速率`：数字输入框，当缓存开关开启时显示
    - 单位：条/秒
    - 范围：1-500
    - 默认：100
- **保存操作**：
  - `保存`：主按钮
    - 点击后按钮变为「保存中...」并禁用
    - 调用保存配置 API
    - 成功后 Toast 提示「保存成功」
- **配置说明**：
  - 缓存开关优先级：网关级 > 平台级
  - 当网关级未配置时，使用平台级默认值（配置入口在「系统管理」→「网关配置」页面）

### 7.5 缓存状态卡片

> 对应需求：AC-008、AC-009、AC-014、AC-015

- **当前配置值**：显示当前生效的缓存配置（网关级或平台级）
- **缓存统计**：
  - `缓存条数`：当前本地缓存的数据条数
  - `存储占用`：已用空间 / 总上限，带进度条
- **补发进度**：
  - 补发进行时显示进度条
  - 格式：已完成 / 总数 + 百分比
  - 补发按采集时间先后顺序（最早的数据先补发）
  - 按配置速率（1-500条/秒）补发
- **清空缓存**：
  - 红色警告按钮
  - 点击后二次确认弹窗
  - 确认后调用清空缓存 API
  - 网关离线时指令缓存，上线后执行
- **历史补发记录**：
  - 显示历史补发的时间、条数、结果

### 7.6 关联设备卡片

- **标题**：关联设备，右上角显示设备总数
- **列表展示**：该网关上已下发的设备实例
- **字段**：设备名称、设备模型、状态、最后数据时间
- **行内操作**：点击设备名称跳转到设备实例详情页

## 10. 业务规则与状态反馈

### 10.1 状态规则

- **心跳频率**：固定每 30 秒一次
- **离线判定**：超过 3 分钟（6 个周期）无心跳视为离线
- **网关状态仅两种**：在线 / 离线
- **状态实时推送**：状态变化通过 SSE 实时推送到前端

### 10.2 缓存开关优先级规则

- 优先级：网关级 > 平台级
- 两级默认均为关闭
- 判断逻辑：
  1. 如果网关级开关有配置值，以网关级配置为准
  2. 否则，使用平台级默认值
- 配置调整后立即生效，无需重新下发配置


### 10.4 通用反馈

- **操作成功**：Toast 提示成功信息，3 秒后消失
- **操作失败**：Toast 或弹窗显示具体错误原因
- **加载状态**：按钮置灰 + 文字变化（如「保存中...」）
- **空状态**：列表为空时显示空状态插画 + 文案
- **搜索无结果**：显示「未找到匹配的网关」
