# 边缘网关管理 — 功能需求说明书

## 1. 背景与价值

**为什么做**：边缘网关是整个系统的基础模块，负责管理分布式部署的 Node-RED 实例。系统需要统一管理这些网关的接入、状态监控、性能监控和断网缓存能力，为上层设备模型管理和配置下发提供运行基础。

**做成什么样**：用户可以在平台上完成网关的注册接入、实时状态监控、分组管理、性能趋势查看，无需逐台登录网关操作。

## 2. 范围

### 本次做

- 平台下发激活（主要方式）：管理员填写 Node-RED 信息 → 平台自动生成 Token → 推送到 Node-RED 配置 adminAuth.tokens → Token 实时验证
- 手动注册（次要方式）：工程师在插件配置注册码 → 插件向平台注册
- 网关列表管理：新增、删除、编辑（名称、缓存开关）
- 测试连接：Node-RED 可达性、MQTT 连接、插件状态三项测试
- 心跳状态监控：在线 / 离线两种状态，3 分钟超时判定，SSE 实时推送
- 断网缓存：两级开关（网关级 > 平台级）控制、网关详情页可配置开关/保存期限/补发速率、状态展示、补发控制
- 性能监控：CPU / 内存 / 磁盘指标上报与历史趋势展示（最长 7 天）
- 网关列表信息：名称、状态、Node-RED 版本、IP、流数、最后心跳时间
- **系统管理模块**：包含注册码管理、网关配置两个页面

### 本次不做

- 设备模型管理 — 另一个功能模块
- 设备实例管理 — 另一个功能模块
- 配置下发与同步 — 另一个功能模块
- 用户权限与角色管理
- 性能告警 — 仅展示指标
- 网关分组管理 — 后续规划

## 4. 需求详述

### 4.1 核心流程

> 这里只描述**主线路径的全局视角**——用户从哪里开始、经过哪些关键节点、到哪里结束。

**平台下发激活流程（主要方式）**
1. 管理员在平台填写网关的 Node-RED 地址、端口、网关名称
2. 点击激活
3. 平台自动生成一个 Token（同时用于平台调用 Node-RED API 和网关身份认证）
4. 平台通过 HTTP 调用 Node-RED Admin API 推送配置：
   - 使用 Token 作为 Basic Auth 凭证
   - 设置 `adminAuth.tokens` 函数，调用平台 API 验证 token
   - 设置 `adminAuth` 启用自定义认证
5. 平台同时推送业务配置（平台地址、MQTT 信息）
6. 激活成功后网关开始上报心跳，状态为在线
7. 后续网关调用 Node-RED Admin API 时，Node-RED 调用平台 API 实时验证 token 有效性

**手动注册流程（次要方式）**
1. 管理员在平台批量生成注册码（设置数量、有效期）
2. 工程师在 Node-RED 上部署 nodered-plugin 插件（插件提前人工部署或从 Node-RED 软件仓库安装）
3. 在插件中配置平台地址、注册码、网关名称
4. 插件向平台发起注册请求，携带注册码和网关信息
5. 平台验证注册码有效性（未使用、未过期、未作废）
6. 验证通过后，平台新增网关记录，注册码标记为已使用，绑定该网关
7. 网关每 30 秒上报一次心跳，平台实时更新状态

**注册码管理流程（位于「系统管理」模块的「注册码」页面）**
1. 管理员进入「系统管理」→「注册码」页面
2. 点击「批量生成」，填写数量（1-50）和有效期
3. 平台生成对应数量的注册码，状态均为「未使用」
4. 管理员可以查看所有注册码的状态（未使用/已使用/已过期/已作废）
5. 点击已使用的注册码可查看绑定的网关详情
6. 未使用的注册码可以直接作废
7. 已使用的注册码可以作废（需二次确认），作废后平台不再接收该网关上报的数据
8. 过期的注册码自动标记为「已过期」

**性能监控流程**
1. 网关每 30 秒上报心跳，消息中附带 CPU / 内存 / 磁盘使用率和剩余空间
2. 平台记录并存储性能历史数据
3. 用户进入网关详情页查看历史趋势图（最长 7 天）
4. 超过 7 天的性能数据自动清理

### 4.2 交互规则

- 删除操作需二次确认弹窗
- 表单提交后按钮置灰防重复提交
- 测试连接结果弹窗显示各项测试结果（通过的项也显示）
- 激活失败时弹窗显示具体失败原因（地址不可达 / 认证失败 / API 调用失败）
- SSE 连接断开时前端自动重试

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 心跳频率固定为每 30 秒一次，超过 3 分钟（6 个周期）无心跳视为离线 |
| BR-002 | 网关状态仅两种：在线 / 离线，有心跳为在线，无心跳为离线 |
| BR-003 | 缓存开关：两级（网关级 > 平台级），默认均为关闭；网关详情页可配置保存期限（默认15天）、补发速率（1-500条/秒，默认100条/秒） |
| BR-004 | 缓存补发：MQTT 重连成功后自动立即补发，严格按采集时间顺序，按配置速率补发 |
| BR-006 | 性能历史数据最长保留 7 天，超过自动清理 |
| BR-007 | 注册码状态分为四种：未使用、已使用、已过期、已作废 |
| BR-008 | 注册码一次最多批量生成 50 个，生成时仅需指定数量和有效期，无需指定网关名称 |
| BR-009 | 注册码作废后，对应网关的数据上报将被平台拒绝 |
| BR-010 | 注册码过期自动标记为「已过期」，过期的注册码保留记录但不可使用 |
| BR-011 | 平台下发激活时，通过 Node-RED Admin API 设置 `adminAuth.tokens` 函数，调用平台 API 实时验证 token |
| BR-012 | Node-RED Admin API 使用平台生成的注册码作为自定义 token，验证通过后网关用户权限为 `read`（只读）|
| BR-013 | 注册码作废后，平台拒绝验证请求，Node-RED 收到 `null` 后拒绝该 token 的 API 请求 |

## 5. 验收标准

> AC 是这份文档的核心交付物。后续的技术方案、任务拆分、TDD 测试用例全部从这里派生。

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 主动注册新网关
  - **Given** 平台上还没有该网关记录，且有一个有效的未使用注册码
  - **When** nodered-plugin 首次注册（含网关唯一ID、名称、注册码）
  - **Then** 平台验证注册码通过，新增一条网关记录，状态为在线，网关列表新增一行，注册码标记为已使用并绑定该网关

- [ ] **AC-002**: 主动注册已有网关
  - **Given** 平台已有该网关记录
  - **When** nodered-plugin 再次上报（网关标识相同）
  - **Then** 平台更新该网关的上报信息（心跳时间、版本、IP、流数、性能指标），不新增记录

- [ ] **AC-003**: 平台下发激活网关
  - **Given** 用户在平台填写了合法的 Node-RED 地址、端口、网关名称
  - **When** 点击激活
  - **Then** 平台自动生成 Token（统一用于平台调用 Node-RED API 和网关身份认证），成功推送 Node-RED 配置（设置 adminAuth.tokens 函数）和业务配置（平台地址、MQTT信息），网关列表新增一行，状态为在线

- [ ] **AC-003a**: 平台下发激活后 Token 验证生效
  - **Given** 平台下发激活成功，Node-RED 已配置 adminAuth.tokens 函数
  - **When** 网关使用 token 调用 Node-RED Admin API
  - **Then** Node-RED 调用平台 API 验证 token，验证通过后允许访问（权限为 read）

- [ ] **AC-003b**: 平台下发激活后 Token 验证拒绝
  - **Given** 平台下发激活成功，注册码被作废
  - **When** 网关使用已作废的 token 调用 Node-RED Admin API
  - **Then** Node-RED 调用平台 API 验证 token，平台返回无效，Node-RED 拒绝该请求

- [ ] **AC-003c**: 平台下发激活失败
  - **Given** 用户在平台填写的 Node-RED 地址无法连接
  - **When** 点击激活
  - **Then** 弹窗报错，提示具体失败原因（地址不可达 / API 调用失败）

- [ ] **AC-004**: 测试连接
  - **Given** 网关在线
  - **When** 用户点击测试连接
  - **Then** 系统依次测试 Node-RED 可达性、MQTT 连接、插件状态，弹窗显示全部测试结果

- [ ] **AC-005**: 心跳超时判定离线
  - **Given** 网关当前在线
  - **When** 超过 3 分钟未收到新的心跳
  - **Then** 网关状态变为离线，前端列表实时刷新

- [ ] **AC-006**: 离线网关恢复心跳
  - **Given** 网关当前离线
  - **When** 收到该网关的心跳
  - **Then** 网关状态变回在线，前端列表实时刷新

- [ ] **AC-007**: 编辑网关
  - **Given** 网关在列表中
  - **When** 用户点击编辑
  - **Then** 用户可以修改网关名称，保存后立即生效

- [ ] **AC-007a**: 编辑网关缓存配置
  - **Given** 用户在网关详情页
  - **When** 用户配置缓存开关、保存期限、补发速率，点击保存
  - **Then** 配置保存成功，立即生效

- [ ] **AC-008**: 断网缓存触发
  - **Given** 网关缓存开关开启（网关级或平台级）
  - **When** 网关与平台网络断开（MQTT 断开或心跳失败）且采集到数据
  - **Then** 数据被写入网关本地缓存

- [ ] **AC-009**: 断网缓存补发
  - **Given** 网关本地有缓存数据，MQTT 重连成功
  - **When** 网络恢复
  - **Then** 系统按采集时间先后顺序、按配置速率（1-500条/秒）补发缓存数据，收到 PUBACK 后清除已发送记录

- [ ] **AC-010**: 网关列表展示
  - **Given** 用户在网关列表查看
  - **Then** 每一行显示网关名称、状态、Node-RED 版本、IP、流数、最后心跳时间

- [ ] **AC-021**: 性能指标上报
  - **Given** 网关在线
  - **When** nodered-plugin 上报心跳（含 CPU 使用率、内存使用率、磁盘使用率、磁盘剩余空间）
  - **Then** 平台记录并存储这些性能指标

- [ ] **AC-022**: 性能趋势查看
  - **Given** 用户在网关列表点击某网关查看详情
  - **Then** 能看到该网关的性能指标趋势图（CPU/内存/磁盘），时间范围可选择（最长 7 天）

- [ ] **AC-023**: 性能数据过期清理
  - **Given** 性能数据超过 7 天保留期
  - **Then** 系统自动清理过期数据

- [ ] **AC-025**: 使用注册码注册网关
  - **Given** 平台存在一个未使用的有效注册码
  - **When** nodered-plugin 使用该注册码发起注册请求
  - **Then** 注册成功，网关记录新增，注册码状态变为「已使用」并绑定该网关

- [ ] **AC-026**: 批量生成注册码
  - **Given** 用户在注册码管理页面
  - **When** 用户输入数量（1-50）和有效期，点击生成
  - **Then** 平台生成对应数量的注册码，状态均为「未使用」，列表中新增对应记录

- [ ] **AC-027**: 批量生成注册码上限校验
  - **Given** 用户在注册码管理页面
  - **When** 用户输入数量超过 50
  - **Then** 提示「单次最多生成 50 个注册码」，生成按钮不可点击

- [ ] **AC-028**: 注册码列表展示
  - **Given** 用户在注册码管理页面
  - **Then** 列表显示注册码、状态、生成时间、过期时间、使用时间、绑定网关名称

- [ ] **AC-029**: 查看已使用注册码的网关信息
  - **Given** 存在一个已使用的注册码
  - **When** 用户点击该注册码
  - **Then** 显示绑定的网关详情（网关名称、注册时间、注册IP）

- [ ] **AC-030**: 注册码过期自动标记
  - **Given** 存在一个未使用的注册码，且已过有效期
  - **When** 系统检查注册码状态
  - **Then** 注册码状态自动变为「已过期」

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-012**: 测试连接部分失败
  - **Given** 测试连接时某一项失败
  - **When** 测试执行
  - **Then** 弹窗显示失败项和错误原因，其他已测项目仍显示结果

- [ ] **AC-013**: 删除网关
  - **Given** 用户删除一个网关
  - **When** 确认删除
  - **Then** 网关记录被移除，已关联的设备实例保留不变

- [ ] **AC-014**: 缓存数据过期清理
  - **Given** 缓存数据超过 15 天
  - **When** 网关检查缓存
  - **Then** 过期数据被自动清理

- [ ] **AC-015**: 缓存达到存储上限
  - **Given** 缓存数据达到物理存储上限
  - **When** 新采集数据需要写入
  - **Then** 停止写入新数据或覆盖最旧数据，以不超过物理上限为准

- [ ] **AC-016**: 缓存配置立即生效
  - **Given** 用户调整网关级或平台级缓存配置（开关/保存期限/补发速率）
  - **When** 保存配置
  - **Then** 新配置立即生效，无需重新下发配置

- [ ] **AC-031**: 使用已过期的注册码注册
  - **Given** 存在一个已过期的注册码
  - **When** nodered-plugin 使用该注册码发起注册请求
  - **Then** 注册失败，返回「注册码已过期」错误

- [ ] **AC-032**: 使用已作废的注册码注册
  - **Given** 存在一个已作废的注册码
  - **When** nodered-plugin 使用该注册码发起注册请求
  - **Then** 注册失败，返回「注册码已作废」错误

- [ ] **AC-033**: 使用已使用的注册码注册
  - **Given** 存在一个已使用的注册码
  - **When** 另一个网关使用该注册码发起注册请求
  - **Then** 注册失败，返回「注册码已被使用」错误

- [ ] **AC-034**: 作废未使用的注册码
  - **Given** 存在一个未使用的注册码
  - **When** 用户点击作废
  - **Then** 注册码状态变为「已作废」，不能再用于注册

- [ ] **AC-035**: 作废已使用的注册码
  - **Given** 存在一个已使用的注册码，对应网关在线
  - **When** 用户点击作废并二次确认
  - **Then** 注册码状态变为「已作废」，平台不再接收该网关上报的数据

- [ ] **AC-036**: 作废已使用的注册码取消
  - **Given** 用户在作废已使用注册码的二次确认弹窗中
  - **When** 用户点击取消
  - **Then** 注册码状态不变，网关数据上报不受影响

- [ ] **AC-037**: 使用不存在的注册码注册
  - **Given** 平台上不存在某注册码
  - **When** nodered-plugin 使用该注册码发起注册请求
  - **Then** 注册失败，返回「注册码不存在」错误

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-018**: 缓存开关优先级 ← BR-003
  - **Given** 任意时刻
  - **When** 判定缓存是否启用
  - **Then** 遵循优先级：网关级开关 > 平台级开关（默认关闭）

- [ ] **AC-019**: 网关状态判定 ← BR-002
  - **Given** 网关状态判定
  - **When** 判断网关状态
  - **Then** 超过 3 分钟无心跳视为离线，有心跳视为在线，状态仅这两种

- [ ] **AC-020**: 缓存补发顺序 ← BR-004
  - **Given** 缓存补发
  - **When** 补发执行
  - **Then** 严格按采集时间先后顺序补发（最早的数据先补发），按配置速率（1-500条/秒）补发

- [ ] **AC-024**: 心跳频率 ← BR-001
  - **Given** 心跳上报
  - **When** 心跳发送
  - **Then** 心跳频率固定为每 30 秒一次，超过 3 分钟（6 个周期）无心跳视为离线

- [ ] **AC-038**: 注册码状态流转 ← BR-007
  - **Given** 任何场景
  - **When** 注册码状态变更
  - **Then** 注册码状态只能是以下四种之一：未使用、已使用、已过期、已作废

- [ ] **AC-039**: 批量生成数量限制 ← BR-008
  - **Given** 批量生成注册码
  - **When** 用户输入数量
  - **Then** 数量范围为 1-50，超出范围不允许生成

- [ ] **AC-040**: 注册码作废效果 ← BR-009
  - **Given** 注册码被作废
  - **When** 对应网关上报数据
  - **Then** 平台拒绝接收数据；未使用的注册码作废后不能再用于注册

- [ ] **AC-041**: 注册码过期规则 ← BR-010
  - **Given** 注册码超过有效期
  - **When** 系统检查状态
  - **Then** 自动标记为「已过期」，保留记录但不可使用

- [ ] **AC-042**: Node-RED Token 实时验证 ← BR-011
  - **Given** 网关使用 token 调用 Node-RED Admin API
  - **When** Node-RED 收到请求
  - **Then** Node-RED 调用平台 API 验证 token，平台返回验证结果

- [ ] **AC-043**: Token 验证通过 ← BR-012
  - **Given** 网关使用有效 token 调用 Node-RED Admin API
  - **When** 平台验证 token 有效
  - **Then** 平台返回验证成功，Node-RED 允许访问，用户权限为 read

- [ ] **AC-044**: Token 验证拒绝 ← BR-013
  - **Given** 网关使用已作废/已过期/已使用的 token 调用 Node-RED Admin API
  - **When** 平台验证 token 无效
  - **Then** 平台返回验证失败，Node-RED 拒绝该请求
