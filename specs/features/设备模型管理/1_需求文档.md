# 设备模型管理 — 功能需求说明书

## 1. 背景与价值

**为什么做**：设备模型管理是整个系统的基础配置模块，负责定义各类工业设备的采集方案模板。用户通过模板定义点位列表和协议参数，基于模板创建设备实例，实现多设备类型的统一管理和快速接入。

**做成什么样**：用户可以快速定义 Modbus / S7 / OPC UA / MQTT / TCP 等各类设备的采集配置，支持版本管理、批量导入导出，降低多设备管理成本。

## 2. 范围

### 本次做

- 设备模板管理：创建、编辑、版本管理、复制、删除
- 点位管理：增删改、批量导入导出（Excel）
- 模板列表：名称、协议、点位数、版本号、更新时间
- 基本信息字段：模型ID、模型名称、协议类型、备注
- 点位信息字段：
  - 公共字段：点位ID、点位名称、描述、数据类型、读写权限、单位、乘系数、偏移量、精度、采集频率、采集超时
  - 协议特殊字段：Modbus（从站地址、功能码、寄存器地址）、S7（DB块号、字节偏移、位偏移）、OPC UA（命名空间索引、节点ID）、MQTT（字段路径JSONPath）、TCP（解析规则表达式）
- 值转换规则：实际值 = (原始值 × 乘系数) + 偏移量
- 版本升级规则：编辑基本信息不升级版本，编辑点位信息才升级版本

### 本次不做

- 设备实例管理 — 另一个功能模块
- 配置下发与同步 — 另一个功能模块
- 点位数据实时展示
- 历史数据查询
- 告警规则配置
- 协议连接配置（IP、端口、串口参数等）— 在设备实例阶段配置

## 4. 需求详述

### 4.1 核心流程

**模板创建与配置流程**
1. 用户点击新建模板，填写基本信息（模型ID、模型名称、协议类型、备注）
2. 保存后模板版本为 v1
3. 在模板详情页进入点位管理，添加点位列表
4. 每个点位填写公共字段和协议专属字段
5. 保存点位后版本号升级（v1 → v2）

**模板版本管理流程**
1. 用户修改模板基本信息（名称、备注等）
2. 保存后不升级版本号
3. 用户修改点位信息（新增、编辑、删除点位）
4. 保存后系统自动递增版本号（v1 → v2 → v3...）
5. 旧版本保留，可查看历史版本
6. 实例绑定的是创建时的模板版本

**模板复制流程**
1. 用户选择已有模板点击复制
2. 新模板名称和 ID 自动加后缀
3. 点位列表与原模板一致
4. 新模板版本为 v1

### 4.2 交互规则

- 删除已有实例关联的模板时，提示"该模板已有实例关联，无法删除"
- 导入点位时如有重复名称，提示重复点位列表
- 选择协议类型后，点位的协议特殊字段区动态变化
- 修改协议类型时，清空原协议特殊字段值并提示重新填写
- 精度字段仅对 FLOAT32/FLOAT64 可见
- S7 协议的位偏移字段仅对 BOOL 类型可见
- Modbus 功能码根据读写权限动态切换选项
  - 只读：01读线圈、02读离散输入、03读保持寄存器、04读输入寄存器
  - 读写：增加 05写单线圈、06写单寄存器、15写多线圈、16写多寄存器

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 点位修改自动产生新版本，版本号从 v1 开始数字递增；基本信息修改不产生新版本 |
| BR-002 | 所有协议点位公共字段包含：点位ID、点位名称、描述、数据类型、读写权限、单位、乘系数、偏移量、精度、采集频率、采集超时 |
| BR-003 | 实际值 = (原始值 × 乘系数) + 偏移量，精度字段控制浮点类型的小数位数 |
| BR-004 | Modbus 点位特殊字段：从站地址（unitid）、功能码（fc）、寄存器地址（address） |
| BR-005 | S7 点位特殊字段：DB块号、字节偏移、位偏移（仅BOOL） |
| BR-006 | OPC UA 点位特殊字段：命名空间索引、节点ID（Node ID） |
| BR-007 | MQTT 点位特殊字段：字段路径（JSONPath），用于从订阅报文中提取该点位值 |
| BR-008 | TCP 点位特殊字段：解析规则表达式，用于从设备上报报文中提取该点位值 |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 创建设备模板
  - **Given** 用户在平台
  - **When** 填写模型ID、模型名称、选择协议类型、填写备注并保存
  - **Then** 模板创建成功，版本号为 v1

- [ ] **AC-002**: 添加点位
  - **Given** 模板已创建
  - **When** 用户添加点位并填写公共字段（点位ID、点位名称、描述、数据类型、读写权限、单位、乘系数、偏移量、精度、采集频率、采集超时）和协议特殊字段
  - **Then** 点位添加成功，显示在点位列表中，模板版本号升级

- [ ] **AC-003**: 修改点位产生新版本
  - **Given** 模板有点位列表
  - **When** 用户修改点位信息（新增、编辑、删除点位）
  - **Then** 自动产生新版本（v1 → v2），旧版本保留

- [ ] **AC-004**: 复制模板
  - **Given** 模板已存在
  - **When** 用户点击复制模板
  - **Then** 新模板创建成功，名称和ID加后缀，点位与原模板一致，版本号为 v1

- [ ] **AC-005**: 选择协议显示对应特殊字段
  - **Given** 模板协议已选定
  - **When** 用户添加点位
  - **Then** 点位表单的协议特殊字段区显示对应协议的专属字段

- [ ] **AC-006**: 导出点位
  - **Given** 模板有点位列表
  - **When** 用户导出点位
  - **Then** 生成 Excel 文件，包含当前版本的所有点位信息

- [ ] **AC-007**: 导入点位
  - **Given** 用户有 Excel 点位文件
  - **When** 导入到模板
  - **Then** 点位批量添加成功，显示在点位列表中，版本号升级

- [ ] **AC-008**: 查看模板列表
  - **Given** 用户在模板列表
  - **When** 查看列表
  - **Then** 每行显示模板名称、协议类型、点位数、版本号、更新时间

- [ ] **AC-009**: 协议特殊字段—Modbus
  - **Given** 模板协议为 Modbus TCP 或 RTU
  - **When** 用户添加点位并填写协议特殊字段（从站地址、功能码、寄存器地址）
  - **Then** 点位添加成功

- [ ] **AC-010**: 协议特殊字段—S7
  - **Given** 模板协议为 S7
  - **When** 用户添加点位并填写协议特殊字段（DB块号、字节偏移、位偏移）
  - **Then** 点位添加成功

- [ ] **AC-011**: 协议特殊字段—OPC UA
  - **Given** 模板协议为 OPC UA
  - **When** 用户添加点位并填写协议特殊字段（命名空间索引、节点ID）
  - **Then** 点位添加成功

- [ ] **AC-012**: 协议特殊字段—MQTT
  - **Given** 模板协议为 MQTT
  - **When** 用户添加点位并填写协议特殊字段（字段路径JSONPath）
  - **Then** 点位添加成功，用于从订阅报文中提取该点位值

- [ ] **AC-013**: 协议特殊字段—TCP
  - **Given** 模板协议为 TCP
  - **When** 用户添加点位并填写协议特殊字段（解析规则表达式）
  - **Then** 点位添加成功，用于从设备上报报文中提取该点位值

- [ ] **AC-014**: 浮点类型显示精度字段
  - **Given** 点位数据类型为 FLOAT32 或 FLOAT64
  - **When** 用户编辑点位
  - **Then** 精度字段可填写

- [ ] **AC-015**: 非浮点类型隐藏精度字段
  - **Given** 点位数据类型为 BOOL 或整型（INT16/UINT16/INT32/UINT32）
  - **When** 用户编辑点位
  - **Then** 精度字段隐藏

- [ ] **AC-016**: 查看历史版本
  - **Given** 模板有多个版本
  - **When** 用户选择查看历史版本
  - **Then** 可以查看各版本的点位列表和基本信息

- [ ] **AC-017**: 模板复制保持点位一致
  - **Given** 模板有点位列表
  - **When** 用户复制模板
  - **Then** 新模板的点位列表与原模板完全一致

- [ ] **AC-018**: 编辑基本信息不升级版本
  - **Given** 模板版本为 v2
  - **When** 用户修改模板的基本信息（名称、备注等）并保存
  - **Then** 版本号保持 v2 不变

- [ ] **AC-019**: Modbus功能码根据读写权限切换
  - **Given** 用户编辑 Modbus 协议点位
  - **When** 切换读写权限
  - **Then** 功能码下拉选项动态变化（只读4个选项，读写8个选项）

- [ ] **AC-020**: S7位偏移仅BOOL可见
  - **Given** 用户编辑 S7 协议点位
  - **When** 数据类型选择 BOOL
  - **Then** 位偏移字段可见；选择其他数据类型时位偏移字段隐藏

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-021**: 删除已有实例的模板
  - **Given** 模板已有实例关联
  - **When** 用户尝试删除模板
  - **Then** 提示"该模板已有实例关联，无法删除"

- [ ] **AC-022**: 导入点位重复
  - **Given** 用户导入点位 Excel
  - **When** 文件中有重复的点位ID
  - **Then** 导入失败，提示重复的点位ID列表

- [ ] **AC-023**: 按协议类型筛选
  - **Given** 用户在模板列表
  - **When** 选择协议类型筛选
  - **Then** 只显示该协议类型的模板

- [ ] **AC-024**: 修改协议类型清空特殊字段
  - **Given** 用户修改模板的协议类型
  - **When** 原协议已有特殊字段值
  - **Then** 清空原协议字段值，提示用户重新填写新协议的参数

- [ ] **AC-025**: 协议必填字段校验
  - **Given** 用户保存点位
  - **When** 协议特殊字段有未填写的必填项
  - **Then** 保存失败，提示"协议必填字段未填写"

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-026**: 版本号递增规则 ← BR-001
  - **Given** 模板版本管理
  - **When** 修改点位信息
  - **Then** 自动产生新版本，版本号从 v1 开始数字递增；修改基本信息不产生新版本

- [ ] **AC-027**: 基本信息字段 ← BR-002
  - **Given** 模板基本信息
  - **Then** 包含：模型ID、模型名称、协议类型、备注

- [ ] **AC-028**: 点位公共字段 ← BR-002
  - **Given** 点位信息
  - **Then** 公共字段包含：点位ID、点位名称、描述、数据类型、读写权限、单位、乘系数、偏移量、精度、采集频率、采集超时

- [ ] **AC-029**: 值转换公式 ← BR-003
  - **Given** 数值类型点位有乘系数和偏移量设置
  - **Then** 采集时应用公式：实际值 = (原始值 × 乘系数) + 偏移量

- [ ] **AC-030**: 精度字段 ← BR-003
  - **Given** 点位数据类型为浮点（FLOAT32/FLOAT64）
  - **When** 展示或存储该点位值
  - **Then** 精度字段生效，用于控制小数位数

- [ ] **AC-031**: Modbus点位字段 ← BR-004
  - **Given** Modbus TCP/RTU 协议
  - **Then** 点位特殊字段：从站地址（unitid）、功能码（fc）、寄存器地址（address）

- [ ] **AC-032**: S7点位字段 ← BR-005
  - **Given** S7 协议
  - **Then** 点位特殊字段：DB块号、字节偏移、位偏移（仅BOOL）

- [ ] **AC-033**: OPC UA点位字段 ← BR-006
  - **Given** OPC UA 协议
  - **Then** 点位特殊字段：命名空间索引、节点ID（Node ID）

- [ ] **AC-034**: MQTT点位字段 ← BR-007
  - **Given** MQTT 协议
  - **Then** 点位特殊字段：字段路径（JSONPath）

- [ ] **AC-035**: TCP点位字段 ← BR-008
  - **Given** TCP 协议
  - **Then** 点位特殊字段：解析规则表达式
