# 设备模型管理 · 需求文档

## 1. 功能概述

设备模型管理负责定义设备类型模板，是设备实例创建的基础。一个设备模型包含两部分信息：

1. **基本信息**（公共字段 + 协议特殊字段）——描述设备的连接方式和协议参数
2. **点位信息**（公共字段 + 协议特殊字段）——描述每个点位的采集地址、数据类型、转换规则等

通过模板化设计，用户可以快速定义各类工业设备的采集方案，并在创建设备实例时继承模板参数，降低多设备类型的管理成本。

## 2. 场景与用户

### 2.1 目标用户

- 工业自动化工程师 — 负责定义设备模板和点位配置
- IoT 系统运维人员 — 负责维护模板版本
- 数据采集系统管理员 — 负责模板管理

### 2.2 典型场景

**场景一：新设备类型接入**
工程师首次接入某型号 Modbus TCP 温湿度传感器，创建设备模板，填写模型基本信息（IP、端口），添加点位（温度、湿度），设置每个点位的寄存器地址、数据类型、乘系数等。

**场景二：模板版本升级**
工程师发现某模板的点位配置有误或需新增点位，修改后模板自动升级到 v2，已创建的实例仍使用 v1，实例管理模块提示"模板有新版本"。

**场景三：模板复制**
工程师需要接入一个相似型号的设备，复制已有模板，修改部分点位和协议参数，快速完成新模板定义。

**场景四：点位批量导入**
工程师从设备厂商获取点位配置 Excel 文件，直接导入到模板，快速完成点位定义，无需逐条手动添加。

**场景五：不同协议设备配置**
工程师需要配置一台西门子 S7 PLC，选择 S7 协议后，填写机架号、槽号等协议特殊字段；配置 OPC UA 服务器时，填写 Endpoint URL、安全策略等字段。

## 3. 核心流程

### 3.1 设备模板管理流程

1. 创建模板：填写基本信息（公共字段 + 协议特殊字段，协议特殊字段随选择的协议类型动态变化）
2. 定义点位列表：添加点位，每个点位包含公共字段 + 协议专属字段
3. 每次修改自动产生新版本（v1 → v2 → v3...），旧版本保留
4. 支持模板复制（名称和ID加后缀，点位一致，版本号从 v1 开始）
5. 支持点位批量导入导出（Excel 格式）

### 3.2 字段结构

模板字段分为两层结构：**基本信息层** 和 **点位信息层**，每层又分 **公共字段** 和 **协议特殊字段**。

```
设备模型
├── 基本信息
│   ├── 公共字段（所有协议相同）
│   │   ├── 模型ID
│   │   ├── 模型名称
│   │   ├── 协议类型
│   │   └── 备注
│   └── 协议特殊字段（随协议类型变化）
│       ├── Modbus TCP：IP、端口
│       ├── Modbus RTU：串口号、波特率、数据位、停止位、校验位
│       ├── S7：IP、机架号、槽号
│       ├── OPC UA：Endpoint URL、安全策略
│       ├── MQTT：Broker地址、端口、用户名、密码、订阅主题、QoS
│       └── TCP：IP、端口、报文分隔符
└── 点位信息
    ├── 公共字段（所有协议相同）
    │   ├── 点位ID
    │   ├── 点位名称
    │   ├── 描述
    │   ├── 数据类型
    │   ├── 读写权限
    │   ├── 单位
    │   ├── 乘系数
    │   ├── 偏移量
    │   ├── 精度（浮点专用）
    │   ├── 采集频率
    │   └── 采集超时
    └── 协议特殊字段（随协议类型变化）
        ├── Modbus：从站地址、功能码、寄存器地址
        ├── S7：DB块号、起始偏移
        ├── OPC UA：命名空间索引、节点ID
        ├── MQTT：字段路径
        └── TCP：解析规则表达式
```

## 4. 详细字段定义

### 4.1 基本信息 — 公共字段（所有协议）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 模型ID | 字符串 | 是 | 模板的唯一标识 |
| 模型名称 | 字符串 | 是 | 模板的显示名称 |
| 协议类型 | 枚举 | 是 | Modbus TCP / Modbus RTU / S7 / OPC UA / MQTT / TCP |
| 备注 | 文本 | 否 | 模板的描述信息 |

### 4.2 基本信息 — 协议特殊字段（随协议变化）

#### Modbus TCP

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| IP地址 | 字符串 | 是 | 目标设备的 IP 地址 |
| 端口 | 整数 | 是 | 默认 502 |

#### Modbus RTU

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 串口号 | 字符串 | 是 | 如 /dev/ttyUSB0、COM1 |
| 波特率 | 枚举 | 是 | 9600 / 19200 / 38400 / 57600 / 115200 |
| 数据位 | 枚举 | 是 | 7 / 8 |
| 停止位 | 枚举 | 是 | 1 / 2 |
| 校验位 | 枚举 | 是 | None / Even / Odd |

#### S7

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| IP地址 | 字符串 | 是 | PLC 的 IP 地址 |
| 机架号 | 整数 | 是 | Rack，通常为 0 |
| 槽号 | 整数 | 是 | Slot，S7-1200/1500 通常为 1，S7-300/400 通常为 2 |

#### OPC UA

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Endpoint URL | 字符串 | 是 | OPC UA 服务器地址，如 opc.tcp://192.168.1.1:4840 |
| 安全策略 | 枚举 | 是 | None / Basic128Rsa15 / Basic256 / Basic256Sha256 / Aes128_Sha256_RsaOaep / Aes256_Sha256_RsaPss |

#### MQTT

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Broker地址 | 字符串 | 是 | MQTT 服务器的 IP 或域名 |
| 端口 | 整数 | 是 | 默认 1883（明文） / 8883（TLS） |
| 用户名 | 字符串 | 否 | 认证用 |
| 密码 | 字符串 | 否 | 认证用 |
| 订阅主题 | 字符串 | 是 | MQTT in 节点订阅的 Topic，支持通配符 |
| QoS | 整数 | 是 | MQTT in 节点的 QoS 配置，传入数值 0/1/2 |

#### TCP

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| IP地址 | 字符串 | 是 | 目标设备的 IP 地址 |
| 端口 | 整数 | 是 | 目标端口 |
| 报文分隔符 | 字符串 | 否 | 用于拆包，如换行符 \n、固定字节数 |

### 4.3 点位信息 — 公共字段（所有协议）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 点位ID | 字符串 | 是 | 点位的唯一标识 |
| 点位名称 | 字符串 | 是 | 点位的显示名称 |
| 描述 | 文本 | 否 | 点位的描述信息 |
| 数据类型 | 枚举 | 是 | BOOL / INT16 / UINT16 / INT32 / UINT32 / FLOAT32 / FLOAT64 / STRING |
| 读写权限 | 枚举 | 是 | 只读 / 只写 / 读写 |
| 单位 | 字符串 | 否 | 如 ℃、%、kPa |
| 乘系数 | 浮点数 | 否 | 默认为 1，用于原始值转换：value = raw * multiplier + offset |
| 偏移量 | 浮点数 | 否 | 默认为 0，用于原始值转换 |
| 精度 | 整数 | 否 | 浮点专用，保留几位小数 |
| 采集频率 | 整数 | 是 | 单位毫秒，如 1000 表示每秒采集一次 |
| 采集超时 | 整数 | 是 | 单位毫秒，超时判定采集失败 |

### 4.4 点位信息 — 协议特殊字段（随协议变化）

#### Modbus（TCP & RTU）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 从站地址 | 整数 | 是 | Slave ID / Unit ID，范围 1-247 |
| 功能码 | 枚举 | 是 | 03 读保持寄存器 / 04 读输入寄存器 / 06 写单个寄存器 / 16 写多个寄存器 |
| 寄存器地址 | 整数 | 是 | 寄存器的起始地址（十进制，如 40001 或直接 0-65535） |

#### S7

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| DB块号 | 整数 | 是 | Data Block 编号，如 DB1 的块号为 1 |
| 起始偏移 | 整数 | 是 | 在 DB 块内的字节偏移，从 0 开始 |

#### OPC UA

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 命名空间索引 | 整数 | 是 | Namespace Index，通常为 0 或 2 |
| 节点ID | 字符串 | 是 | Node ID，如 ns=2;s=Temperature 或 ns=3;i=1001 |

#### MQTT

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 字段路径 | 字符串 | 是 | 从订阅报文中提取该点位值的 JSON Path 或字段名，如 `data.temperature` 或 `temperature` |

#### TCP

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 解析规则表达式 | 字符串 | 是 | 描述从报文中提取该点位值的规则，支持键值对格式（如 `key=temp` 提取 `temp=25.6` 中的 25.6）或正则表达式（如 `T(\d+\.\d+)` 提取小数部分） |

### 4.5 值转换规则

对于数值类型的点位，实际值计算公式为：

```
实际值 = (原始值 × 乘系数) + 偏移量
```

- 乘系数默认为 1
- 偏移量默认为 0
- 浮点数据类型使用精度字段控制小数位数
- BOOL 类型不参与乘系数和偏移量转换

## 5. 边界与异常处理

### 5.1 模板管理

| 异常场景 | 处理方式 |
|---------|---------|
| 删除已有实例关联的模板 | 提示"该模板已有实例关联，无法删除" |
| 导入点位时点位ID重复 | 导入失败，提示重复的点位ID列表 |
| 模板复制时名称重复 | 名称和ID自动加后缀 |

### 5.2 协议字段

| 异常场景 | 处理方式 |
|---------|---------|
| 修改协议类型后已有协议特殊字段 | 清空原协议的特殊字段值，提示用户重新填写 |
| 点位的协议特殊字段为空 | 保存时校验并提示"协议必填字段未填写" |

## 6. 业务规则

### 6.1 版本管理规则

- 每次修改模板（基本信息或点位信息）自动产生新版本
- 版本号从 v1 开始，数字递增（v1 → v2 → v3...）
- 旧版本保留，可查看历史版本

### 6.2 分组规则

- 模板分组：按协议分组（Modbus类、S7类、OPC UA类、MQTT类、TCP类）
- 可自定义业务分组

### 6.3 字段联动规则

- 选择协议类型后：基本信息的协议特殊字段区动态变化
- 选择数据类型后：精度字段仅对 FLOAT32/FLOAT64 可见
- 读写权限为"只读"时：不显示写入相关的协议字段

### 6.4 MQTT / TCP 特殊规则

- MQTT 和 TCP 均为设备主动推送模式：设备上报报文结构由设备决定，点位无法预定义固定地址
- **MQTT 点位**：使用"字段路径"从订阅报文中动态提取值，支持 JSON Path 语法（如 `data.temperature`）或根级字段名（如 `temperature`）
- **TCP 点位**：使用"解析规则表达式"从报文中动态提取值，支持键值对格式（如 `key=temp` 提取 `temp=25.6` 中的 25.6）或正则表达式（如 `T(\d+\.\d+)` 提取小数部分）
- MQTT 基本信息中的订阅主题和 QoS 统一配置在 MQTT in 节点上，所有点位共用同一订阅配置

## 7. 范围界定

### 7.1 本次做

- 设备模板管理：创建、编辑、版本管理、复制、删除、分组筛选
- 基本信息字段：公共字段（模型ID、模型名称、协议类型、备注）+ 6种协议的协议特殊字段
- 点位信息字段：公共字段（点位ID、名称、描述、数据类型、读写权限、单位、乘系数、偏移量、精度、采集频率、采集超时）+ 5种协议的点位特殊字段
- 点位管理：增删改、批量导入导出（Excel）
- 值转换规则：乘系数 + 偏移量
- 模板列表：名称、协议、分组、点位数、版本号、创建时间

### 7.2 本次不做

- 设备实例管理（独立功能模块）
- 配置下发与同步（独立功能模块）
- 点位数据实时展示
- 历史数据查询
- 告警规则配置

## 8. 验收标准（AC）

### 8.1 Happy Path

| 编号 | 验收标准 |
|------|---------|
| AC-001 | Given 用户创建设备模板，When 填写公共字段（模型ID、模型名称、协议类型、备注）并保存，Then 模板创建成功，版本号为 v1。 |
| AC-002 | Given 用户选择协议类型为 Modbus TCP，Then 基本信息的协议特殊字段显示 IP地址、端口。 |
| AC-003 | Given 用户选择协议类型为 Modbus RTU，Then 基本信息的协议特殊字段显示串口号、波特率、数据位、停止位、校验位。 |
| AC-004 | Given 用户选择协议类型为 S7，Then 基本信息的协议特殊字段显示 IP地址、机架号、槽号。 |
| AC-005 | Given 用户选择协议类型为 OPC UA，Then 基本信息的协议特殊字段显示 Endpoint URL、安全策略。 |
| AC-006 | Given 用户选择协议类型为 MQTT，Then 基本信息的协议特殊字段显示 Broker地址、端口、用户名、密码、订阅主题、QoS。 |
| AC-007 | Given 用户选择协议类型为 TCP，Then 基本信息的协议特殊字段显示 IP地址、端口、报文分隔符。 |
| AC-008 | Given 模板已创建，When 用户添加点位并填写公共字段（点位ID、名称、描述、数据类型、读写权限、单位、乘系数、偏移量、精度、采集频率、采集超时），Then 点位添加成功。 |
| AC-009 | Given 模板协议为 Modbus，When 用户添加点位并填写协议特殊字段（从站地址、功能码、寄存器地址），Then 点位添加成功。 |
| AC-010 | Given 模板协议为 S7，When 用户添加点位并填写协议特殊字段（DB块号、起始偏移），Then 点位添加成功。 |
| AC-011 | Given 模板协议为 OPC UA，When 用户添加点位并填写协议特殊字段（命名空间索引、节点ID），Then 点位添加成功。 |
| AC-012 | Given 模板协议为 MQTT，When 用户添加点位并填写协议特殊字段（字段路径），Then 点位添加成功，用于从订阅报文中提取该点位值。 |
| AC-013 | Given 模板协议为 TCP，When 用户添加点位并填写协议特殊字段（解析规则表达式），Then 点位添加成功，用于从设备上报报文中提取该点位值。 |
| AC-014 | Given 点位数据类型为 FLOAT32 或 FLOAT64，Then 精度字段可填写，用于控制小数位数。 |
| AC-015 | Given 点位数据类型为 BOOL 或整型（INT16/UINT16/INT32/UINT32），Then 精度字段隐藏或禁用。 |
| AC-016 | Given 模板有点位列表，When 用户修改模板内容（基本信息或点位），Then 自动产生新版本（v1 → v2），旧版本保留。 |
| AC-017 | Given 模板已存在，When 用户点击复制模板，Then 新模板创建成功，名称和ID加后缀，点位与原模板一致，版本号为 v1。 |
| AC-018 | Given 模板点位列表，When 用户导出点位，Then 生成 Excel 文件，包含当前版本所有点位信息（公共字段 + 协议特殊字段）。 |
| AC-019 | Given 用户有 Excel 点位文件，When 导入到模板，Then 点位批量添加成功，显示在点位列表中。 |
| AC-020 | Given 用户在模板列表查看，Then 每一行显示名称、协议、分组、点位数、版本号、创建时间。 |
| AC-021 | Given 点位设置了乘系数和偏移量，Then 数据采集时应用公式：实际值 = (原始值 × 乘系数) + 偏移量。 |
| AC-022 | Given 点位读写权限为只读，Then 协议的写入相关字段（如功能码 06/16、发布主题）隐藏或禁用。 |

### 8.2 Edge & Error Cases

| 编号 | 验收标准 |
|------|---------|
| AC-023 | Given 模板已有实例关联，When 用户尝试删除模板，Then 提示"该模板已有实例关联，无法删除"。 |
| AC-024 | Given 用户导入点位 Excel，When 文件中有重复的点位ID，Then 导入失败，提示重复的点位ID列表。 |
| AC-025 | Given 用户在模板列表按协议分组筛选，Then 只显示该协议分组下的模板。 |
| AC-026 | Given 用户修改模板的协议类型，When 原协议已有特殊字段值，Then 清空原协议字段值，提示用户重新填写新协议的参数。 |
| AC-027 | Given 用户保存点位时，When 协议特殊字段有未填写的必填项，Then 保存失败，提示"协议必填字段未填写"。 |

### 8.3 Business Rules

| 编号 | 验收标准 |
|------|---------|
| AC-028 | Given 模板版本管理，Then 每次修改自动产生新版本，版本号从 v1 开始数字递增。 |
| AC-029 | Given 点位基本信息，Then 公共字段包含：模型ID、模型名称、协议类型、备注。 |
| AC-030 | Given 点位信息，Then 公共字段包含：点位ID、名称、描述、数据类型、读写权限、单位、乘系数、偏移量、精度、采集频率、采集超时。 |
| AC-031 | Given 不同协议的模板，Then 基本信息和点位信息各协议有独立的特殊字段（Modbus TCP/RTU、S7、OPC UA、MQTT、TCP）。 |
| AC-032 | Given 数值类型点位有乘系数和偏移量设置，Then 采集时自动应用转换公式。 |
| AC-033 | Given 点位数据类型为浮点（FLOAT32/FLOAT64），Then 精度字段生效，用于控制小数位数。 |
