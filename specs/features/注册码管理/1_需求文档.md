# 注册码管理 — 功能需求说明书

## 1. 背景与价值

**为什么做**：注册码是网关接入平台的核心凭证，用于验证网关身份、控制接入权限、追踪网关使用情况。通过统一管理注册码生命周期（生成→使用→作废→删除），平台可以安全、可控地管理网关接入。

**做成什么样**：管理员可以批量生成注册码，追踪使用情况，作废无效注册码；平台下发激活时自动生成 Token，统一认证体系。

---

## 2. 范围

### 本次做

- 注册码生成：HMAC-SHA256 算法，支持批量生成（最多 50 个）
- 注册码管理页面：列表展示、搜索、分页
- 注册码状态管理：未使用、已使用、已过期、已作废
- 作废操作：未使用的直接作废，已使用的需二次确认后再作废
- 删除操作：只有已作废的注册码才能删除

### 本次不做

- 注册码导出（Excel/CSV）— 后续规划
- 注册码批量作废 — 后续规划
- 注册码使用统计报表 — 后续规划

### 当前调整（CR-001）

- 暂时隐藏前端注册码管理入口，不再向运维人员开放注册码列表页、批量生成弹窗和相关页面路由
- 后端 registration API、数据库模型和 nodered-plugin 注册校验保持兼容，避免影响现有网关注册流程

## 4. 需求详述

### 4.1 核心流程

> 这里只描述**主线路径的全局视角**——用户从哪里开始、经过哪些关键节点、到哪里结束。

**注册码生成流程**
1. 管理员进入「系统管理」→「注册码」页面
2. 点击「批量生成」，填写数量（1-50）和有效期
3. 平台用 HMAC-SHA256 算法生成注册码，存入数据库（仅存盐值，不存明文）
4. 注册码列表显示新生成的记录，状态为「未使用」

**注册码验证流程（由 nodered-plugin 触发）**
1. 工程师在 nodered-plugin 中配置平台地址和注册码
2. 插件向平台发起验证请求
3. 平台用盐值 + 密钥重新计算签名，比对验证
4. 验证通过：新增网关记录，注册码标记为「已使用」并绑定网关
5. 验证失败：返回具体错误原因（已过期 / 已作废 / 已被使用 / 不存在）

### 4.2 交互规则

- 表单提交后按钮置灰防重复提交
- 删除操作需二次确认弹窗
- 列表加载时显示加载状态
- 搜索或筛选条件变化时自动刷新列表
- 已使用注册码作废需二次确认弹窗

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 注册码格式：`{随机数前8位}-{签名后8位}`，如 `A1B2C3D4-E5F6G7H8` |
| BR-002 | 注册码使用 HMAC-SHA256 算法生成，平台仅存储盐值（salt）和密钥（secret），不存储明文注册码 |
| BR-003 | 有效期预设选项：7天 / 30天 / 90天 / 180天 / 永不过期 / 自定义（1-3650天） |
| BR-004 | 注册码状态：未使用 → 已使用 / 已过期 / 已作废；已使用 → 已作废；已作废 → 可删除；已过期不可逆 |
| BR-005 | 列表搜索支持：注册码内容模糊搜索、网关名称模糊搜索、状态筛选 |
| BR-006 | 列表默认按生成时间倒序排列，每页默认 20 条，支持切换 10/20/50 条/页 |
| BR-007 | 注册码列表显示格式：`A1B2C3D4-********`（后8位隐藏） |
| BR-008 | 注册码可被两种方式使用：平台下发激活（自动生成）、手动注册（管理员预生成） |

## 5. 验收标准

> AC 是这份文档的核心交付物。后续的技术方案、任务拆分、TDD 测试用例全部从这里派生。

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 批量生成注册码
  - **Given** 管理员在注册码管理页面
  - **When** 输入数量（50）和有效期（30天），点击生成
  - **Then** 平台生成 50 个注册码，列表新增 50 行，状态均为「未使用」

- [ ] **AC-002**: 生成数量上限校验
  - **Given** 管理员在注册码管理页面
  - **When** 输入数量超过 50
  - **Then** 提示「单次最多生成 50 个注册码」，生成按钮置灰

- [ ] **AC-003**: 生成有效期默认值
  - **Given** 管理员在注册码管理页面
  - **When** 打开生成弹窗
  - **Then** 有效期默认选中「30天」

- [ ] **AC-004**: 注册码列表展示
  - **Given** 管理员在注册码管理页面
  - **Then** 列表显示：注册码（前8位+后8位隐藏）、状态、生成时间、过期时间、使用时间（已使用时）、绑定网关（已使用时）、操作按钮

- [ ] **AC-005**: 注册码搜索
  - **Given** 管理员在注册码管理页面
  - **When** 输入注册码内容 / 网关名称 / 选择状态进行筛选
  - **Then** 列表实时过滤显示匹配结果

- [ ] **AC-006**: 作废未使用的注册码
  - **Given** 存在一个未使用的注册码
  - **When** 管理员点击「作废」
  - **Then** 注册码状态变为「已作废」，操作按钮变为「删除」

- [ ] **AC-007**: 作废已使用的注册码
  - **Given** 存在一个已使用的注册码，对应网关在线
  - **When** 管理员点击「作废（需确认）」，在弹窗中二次确认
  - **Then** 注册码状态变为「已作废」，平台拒绝该网关后续的数据上报

- [ ] **AC-008**: 作废已使用的注册码取消
  - **Given** 管理员在作废已使用注册码的二次确认弹窗中
  - **When** 点击取消
  - **Then** 注册码状态不变，网关数据上报不受影响

- [ ] **AC-009**: 删除已作废的注册码
  - **Given** 存在一个已作废的注册码
  - **When** 管理员点击「删除」，在弹窗中确认
  - **Then** 注册码从列表移除

- [ ] **AC-010**: 删除未使用的注册码
  - **Given** 存在一个未使用的注册码
  - **When** 管理员尝试删除
  - **Then** 删除按钮不存在，操作按钮仅为「作废」

- [ ] **AC-011**: 查看已使用注册码的网关信息
  - **Given** 存在一个已使用的注册码
  - **When** 管理员点击该注册码
  - **Then** 弹出详情显示：绑定网关名称、注册时间、注册IP

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-012**: 使用已过期的注册码
  - **Given** 存在一个已过期的注册码
  - **When** nodered-plugin 使用该注册码发起验证请求
  - **Then** 返回错误「注册码已过期」，注册码状态不变

- [ ] **AC-013**: 使用已作废的注册码
  - **Given** 存在一个已作废的注册码
  - **When** nodered-plugin 使用该注册码发起验证请求
  - **Then** 返回错误「注册码已作废」，注册码状态不变

- [ ] **AC-014**: 使用已使用的注册码
  - **Given** 存在一个已使用的注册码
  - **When** 另一个网关使用该注册码发起验证请求
  - **Then** 返回错误「注册码已被使用」，注册码状态不变

- [ ] **AC-015**: 使用不存在的注册码
  - **Given** 平台上不存在某注册码
  - **When** nodered-plugin 使用该注册码发起验证请求
  - **Then** 返回错误「注册码不存在」

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-016**: 批量生成数量限制 ← BR-005
  - **Given** 管理员输入生成数量
  - **When** 数量超出 1-50 范围
  - **Then** 提示错误信息，生成按钮置灰

- [ ] **AC-017**: 自定义有效期范围 ← BR-003
  - **Given** 管理员选择「自定义」有效期
  - **When** 输入天数不在 1-3650 范围
  - **Then** 提示「有效期需在 1-3650 天之间」，提交按钮置灰

- [ ] **AC-018**: 注册码加密算法 ← BR-002
  - **Given** 注册码生成或验证时
  - **When** 系统处理注册码
  - **Then** 使用 HMAC-SHA256 算法，平台仅存储 salt 不存储明文

- [ ] **AC-019**: 注册码状态终态 ← BR-004
  - **Given** 注册码状态为已使用、已过期或已作废
  - **When** 尝试将状态改回「未使用」
  - **Then** 不允许，状态不可逆

- [ ] **AC-020**: 已作废后可删除 ← BR-004
  - **Given** 注册码状态为已作废
  - **When** 管理员请求删除
  - **Then** 允许删除，列表移除该记录

- [ ] **AC-021**: 注册码不可重复使用 ← BR-004
  - **Given** 注册码状态为已使用、已过期或已作废
  - **When** 再次使用该注册码验证
  - **Then** 返回对应错误，注册码状态保持不变
