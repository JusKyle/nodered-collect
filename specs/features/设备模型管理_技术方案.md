## 2. 数据模型设计

### 2.1 Prisma Schema

```prisma
model DeviceModel {
  id          String        @id @default(cuid())
  name        String
  vendor      String
  model       String
  protocol    String
  description String?
  status      ModelStatus   @default(ENABLED)
  points      Json          @default("[]")
  version     String        @default("v1.0")
  createdAt   DateTime      @default(now())
  updatedAt   DateTime      @updatedAt

  deviceInstances DeviceInstance[]
  modelVersions   ModelVersion[]

  @@unique([name, vendor, model])
  @@index([status])
}

model ModelVersion {
  id          String      @id @default(cuid())
  modelId     String
  version     String
  points      Json
  changelog   String?
  createdAt   DateTime    @default(now())

  model       DeviceModel @relation(fields: [modelId], references: [id])

  @@unique([modelId, version])
}

enum ModelStatus {
  ENABLED
  DISABLED
}
```

### 2.2 Point 数据结构

```typescript
interface Point {
```

## 3. API 设计

### 3.1 设备模型 API

#### GET /api/device-models
获取模型列表。

**响应**
```json
{
  "success": true,
  "data": [
    {
      "id": "clx123...",
      "name": "西门子 S7-1200",
      "vendor": "西门子",
      "model": "S7-1200",
      "protocol": "S7",
      "description": "标准 PLC",
      "status": "ENABLED",
      "points": [...],
      "version": "v1.2",
      "pointCount": 15,
      "usageCount": 3,
      "createdAt": "2026-06-17T10:30:00Z"
    }
  ]
}
```

#### POST /api/device-models
创建模型。

**请求**
```json
{
  "name": "西门子 S7-1200",
  "vendor": "西门子",
  "model": "S7-1200",
  "protocol": "S7",
  "description": "标准 PLC",
  "points": [...]
}
```

#### PUT /api/device-models/:id
更新模型（创建新版本）。

**行为**：
1. 获取当前版本号（如 v1.0）
2. 将当前点位存入 ModelVersion
3. 更新 DeviceModel 的 points 和 version
4. 返回新版本号

#### POST /api/device-models/:id/duplicate
复制模型。

**响应**
```json
{
  "success": true,
  "data": {
    "id": "clx456...",
    "name": "西门子 S7-1200_副本",
    "vendor": "西门子",
    "model": "S7-1200_副本",
    "version": "v1.0",
    "pointCount": 15
  }
}
```

#### PUT /api/device-models/:id/status
切换状态。

**请求**
```json
{ "status": "DISABLED" }
```

#### DELETE /api/device-models/:id
删除模型。

**行为**：检查 `usageCount > 0` 则返回错误。

#### POST /api/device-models/:id/points/import
导入点位。

**请求**：FormData
```json
{
  "points": [
    { "name": "温度", "code": "temp", ... },
    ...
  ]
}
```

## 4. 核心逻辑设计

### 4.1 版本递增逻辑

```typescript
// device-model.service.ts
async updateModel(id: string, dto: UpdateDeviceModelDto) {
  const model = await prisma.deviceModel.findUnique({ where: { id } });

  // 1. 保存当前版本到历史
  await prisma.modelVersion.create({
    data: {
      modelId: id,
      version: model.version,
      points: model.points,
    }
  });

  // 2. 递增版本号
  const versionParts = model.version.replace('v', '').split('.');
  const newVersion = `v${versionParts[0]}.${parseInt(versionParts[1]) + 1}`;

  // 3. 更新模型
  return prisma.deviceModel.update({
    where: { id },
    data: { ...dto, version: newVersion }
  });
}
```

→ AC-003: 版本管理

### 4.2 点位编码唯一性校验

```typescript
// device-model.dto.ts
export const createDeviceModelDto = z.object({
  // ...
  points: z.array(z.object({
    name: z.string(),
    code: z.string()
  }))
}).refine(
  (data) => {
    const codes = data.points?.map(p => p.code) || [];
    return new Set(codes).size === codes.length;
  },
  { message: "点位编码不能重复" }
);
```

→ AC-002: 点位管理
