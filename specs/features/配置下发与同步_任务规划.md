## 3. 任务清单

### 切片 1: 同步记录列表

**完成标准**: 用户打开同步记录页面，显示下发历史，支持多条件筛选。

| 任务编号 | 任务描述 | 通俗解释 | 验证标准 | AC 编号 |
|----------|----------|----------|----------|----------|
| T1 | 创建 SyncRecords 页面 | 用户打开同步记录页面时，显示下发历史列表 | 页面加载 → 显示同步记录列表，按下发时间倒序排列 | AC-004 |
| T2 | 实现搜索筛选 | 用户可以按网关、实例、结果、时间范围筛选 | 选择网关"产线1" → 只显示该网关的记录<br>选择结果"失败" → 只显示失败记录<br>选择时间范围 → 只显示该时间范围内的记录 | AC-004 |
| T3 | 实现分页功能 | 用户可以翻页查看更多记录 | 点击下一页 → 加载下一页数据<br>显示总条数和当前页 | AC-004 |
| T4 | 实现行内操作按钮 | 显示查看详情按钮 | 每行末尾显示"查看详情"按钮 | AC-004 |

### 切片 2: 下发详情

**完成标准**: 用户可以查看下发详情，包括下发内容、返回结果和重试记录。

| 任务编号 | 任务描述 | 通俗解释 | 验证标准 | AC 编号 |
|----------|----------|----------|----------|----------|
| T5 | 创建 DispatchLogDetailModal 组件 | 用户点击"查看详情"按钮时，弹出详情窗口 | 点击按钮 → 弹窗打开，显示下发时间、网关、实例、操作类型、结果 | AC-004 |
| T6 | 实现详情展示 | 用户可以查看下发内容 JSON 和重试记录 | 下发内容显示 JSON 格式，可折叠/展开<br>返回结果显示 JSON 格式，可折叠/展开<br>重试记录显示时间、结果、原因 | AC-004 |

### 切片 3: 下发配置

**完成标准**: 用户下发配置到网关，失败自动重试最多3次。

| 任务编号 | 任务描述 | 通俗解释 | 验证标准 | AC 编号 |
|----------|----------|----------|----------|----------|
| T7 | 创建 DispatchConfirmBubble 组件 | 用户点击"下发配置"按钮时，弹出确认气泡 | 点击按钮 → 气泡弹出，显示"确定下发配置到网关「XXX」吗？" | AC-001, AC-002 |
| T8 | 实现下发逻辑 | 用户确认后，调用下发 API | 点击"确定" → POST /sync/dispatch<br>下发中 → 按钮置灰显示"下发中..."<br>下发成功 → 状态变为"运行中"，列表新增记录 | AC-001, AC-002 |
| T9 | 实现失败重试提示 | 下发失败后显示错误原因 | 下发失败 → 顶部红色提示失败原因<br>自动重试 3 次（10s → 30s → 60s）<br>重试期间列表显示"重试中"状态 | AC-003 |

### 切片 4: 解除下发

**完成标准**: 用户可以解除下发，设备状态变为"待同步"。

| 任务编号 | 任务描述 | 通俗解释 | 验证标准 | AC 编号 |
|----------|----------|----------|----------|----------|
| T10 | 创建 UndispatchConfirmBubble 组件 | 用户点击"解除下发"按钮时，弹出确认气泡 | 点击按钮 → 气泡弹出，显示"确定解除下发？" | AC-001 |
| T11 | 实现解除下发功能 | 用户确认后，调用解除 API | 点击"确定" → POST /sync/undeploy<br>解除成功 → 状态变为"待同步" | AC-001 |

### 切片 5: 同步状态面板

**完成标准**: 用户可以查看网关的同步状态和缓存补发进度。

| 任务编号 | 任务描述 | 通俗解释 | 验证标准 | AC 编号 |
|----------|----------|----------|----------|----------|
| T12 | 创建 SyncStatusPanel 组件 | 用户在网关详情页查看同步状态面板 | 页面显示：网关状态、MQTT连接状态、最近上报时间、今日上报条数 | AC-005, AC-006 |
| T13 | 实现缓存进度显示 | 断网期间显示缓存数据条数 | 网关离线 → 显示"缓存数据条数：X"<br>网关恢复 → 显示"正在补发 X/Y 条"，进度条实时更新 | AC-006 |
| T14 | 实现补发状态显示 | 用户可以看到补发进度和结果 | 补发中 → 显示进度条和"补发中"<br>补发完成 → 显示"已完成"<br>部分失败 → 显示"部分失败，X 条失败" | AC-006 |

### 切片 6: Node-RED 端数据缓存

**完成标准**: 断网期间数据缓存到 SQLite，网络恢复后自动补发。

| 任务编号 | 任务描述 | 通俗解释 | 验证标准 | AC 编号 |
|----------|----------|----------|----------|----------|
| T15 | 实现 SQLite 数据缓存 | 断网时采集数据写入本地 SQLite | 断网检测 → 采集数据写入 data_cache 表<br>包含字段：deviceId, pointCode, value, timestamp, quality | AC-005 |
| T16 | 实现自动补发 | 网络恢复后按时间顺序补发缓存数据 | 网络恢复 → 自动从 SQLite 读取缓存数据<br>批量发送到后端 → 确认后删除已补发数据<br>显示补发进度 | AC-006 |
