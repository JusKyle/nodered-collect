# 项目开发路线图 (Development Roadmap)

## 0. 策略概览

*   **开发策略**: 先搭地基（基础设施 + 网关管理），再做核心业务（设备模型 + 实例 + 配置下发），最后完善体验（实时推送 + 断网缓存 + 性能监控 + 历史数据）。
*   **MVP 目标**: 完成从网关注册 → 设备模板定义 → 设备实例创建 → 配置下发 → 数据采集上报的完整闭环，用户可以在平台上看到实时采集值。
*   **当前进度**: 里程碑一（MVP）已全部完成，包含网关管理、设备模型、设备实例、配置下发、数据采集、SSE 实时推送、历史数据查询等核心功能；里程碑二正在进行中，网关侧断网缓存已完成，平台侧缓存管理和性能监控待开发。

---

## 1. 里程碑一：MVP (核心闭环)
> 目标：实现最基础的业务流程，完成从 0 到 1 的闭环。

### Phase 1.1: 基础设施 (Infrastructure)

- [x] **后端基础框架**
    *   **模块**: Express + TypeScript + Prisma + 中间件（auth/error/validation）
    *   **说明**: 后端服务骨架、路由聚合、配置管理（db/env/mqtt/redis）、统一响应格式、错误处理。

- [x] **前端基础框架**
    *   **模块**: React + Vite + TypeScript + TailwindCSS + Zustand
    *   **说明**: 前端项目骨架、MainLayout 布局、Sidebar 侧边栏、路由配置、axios 实例、toast 工具。

- [x] **数据库 Schema**
    *   **模块**: Prisma schema.prisma
    *   **说明**: Gateway / DeviceModel / PointModel / DeviceInstance / SyncRecord 等核心表结构。

- [x] **认证体系**
    *   **模块**: JWT + bcrypt + TokenService + 认证中间件
    *   **说明**: 用户注册登录（后端已支持）、Token 校验中间件。

### Phase 1.2: 边缘网关管理 (Gateway Management)

- [x] **网关注册接入**
    *   **模块**: gateway / registration 模块
    *   **说明**: nodered-plugin 主动注册 + 平台手动添加网关 + 注册码生成。

- [x] **网关列表管理**
    *   **模块**: gateway 模块 + 前端 GatewayList 页面
    *   **说明**: 网关 CRUD、分组管理、状态展示（在线/离线）、测试连接。

- [x] **心跳检测**
    *   **模块**: heartbeat.service.ts + mqtt.service.ts
    *   **说明**: 30 秒心跳、3 分钟超时判定离线、Redis 心跳 key 机制。

### Phase 1.3: 设备模型管理 (Device Model)

- [x] **设备模板管理**
    *   **模块**: device-model 模块 + 前端 DeviceModelList 页面
    *   **说明**: 模板 CRUD、版本管理（v1/v2...）、模板复制、协议类型切换。

- [x] **点位管理**
    *   **模块**: PointModel + device-model service
    *   **说明**: 点位 CRUD、公共字段（名称/数据类型/单位/乘系数/偏移量/精度/采集频率）、协议特殊字段。

- [x] **批量导入导出**
    *   **模块**: device-model 模块 + ImportPointsModal
    *   **说明**: Excel 导入点位、导出当前版本点位。

### Phase 1.4: 设备实例管理 (Device Instance)

- [x] **实例创建与配置**
    *   **模块**: device-instance 模块 + 前端 DeviceInstanceList 页面
    *   **说明**: 从模板创建实例、通信参数配置、网关分配、点位继承。

- [x] **点位继承与覆盖**
    *   **模块**: device-instance service
    *   **说明**: 继承模板点位、修改继承点位属性、新增设备级点位。

- [x] **同步模板版本**
    *   **模块**: device-instance 模块 + SyncPointsConfirmBubble
    *   **说明**: 模板有新版本时提示、同步到最新版本、保留设备级点位。

### Phase 1.5: 配置下发与同步 (Sync & Dispatch)

- [x] **配置下发**
    *   **模块**: sync 模块 + dispatch.service.ts + 前端 SyncRecords 页面
    *   **说明**: 生成 Node-RED Flow、调用 Node-RED /flows API 部署、单个/批量下发。

- [x] **版本比对**
    *   **模块**: sync service
    *   **说明**: 配置版本号与已下发版本号比对、有变化才下发、无变化提示跳过。

- [x] **下发历史**
    *   **模块**: SyncRecord 表 + 前端 DispatchLogDetailModal
    *   **说明**: 记录每次下发结果、失败原因、下发类型（新增/更新/删除）。

### Phase 1.6: 数据采集与实时推送 (Data Collection & SSE)

- [x] **MQTT 数据接入**
    *   **模块**: mqtt.service.ts + data-collection.service.ts + device-data 模块
    *   **说明**: 订阅网关数据上报 Topic、数据解析、写入时序库。

- [x] **SSE 实时推送**
    *   **模块**: sse.service.ts + 前端 useGatewaySSE Hook
    *   **说明**: 网关状态变化推送、采集值实时推送、下发状态推送。

- [x] **实时值展示**
    *   **模块**: 前端 DeviceDataPanel + PointValueTable
    *   **说明**: 实例详情页点位实时值展示、SSE 更新。

## 2. 里程碑二：功能完善 (Feature Complete)
> 目标：补充边缘场景和监控能力，提升产品完整性和用户体验。

### Phase 2.1: 性能监控 (Performance Monitoring)

- [ ] **网关心跳携带性能指标**
    *   **模块**: heartbeat 服务 + nodered-plugin
    *   **说明**: 心跳 payload 增加 CPU 使用率、内存使用率、磁盘使用率、磁盘剩余空间。

- [ ] **性能数据存储**
    *   **模块**: 后端数据采集服务
    *   **说明**: 性能数据存入时序库、7 天自动清理策略。

- [ ] **网关详情页**
    *   **模块**: 前端 GatewayDetail 页面
    *   **说明**: 性能趋势图（CPU/内存/磁盘）、缓存状态信息、关联设备列表。

### Phase 2.2: 断网数据缓存 (Offline Cache)

- [x] **网关侧缓存基础能力**
    *   **模块**: nodered-plugin (data-cache.service + sqlite-store)
    *   **说明**: SQLite 本地存储、数据缓存写入、网络恢复后批量补发、MQTT 补发 Topic。

- [x] **数据库 Schema 支持**
    *   **模块**: Prisma schema (Gateway.cacheEnabled / DeviceInstance.cacheEnabled)
    *   **说明**: 网关级缓存开关字段、设备级覆盖开关字段，为平台侧管理提供数据基础。

- [ ] **三级缓存开关管理**
    *   **模块**: gateway / device-instance / 系统设置
    *   **说明**: 系统级/网关级/设备级开关页面、优先级规则（设备级 > 网关级 > 系统级）、默认关闭。

- [ ] **缓存状态展示**
    *   **模块**: 网关列表 + 网关详情页
    *   **说明**: 列表显示缓存条数、详情页显示存储空间、补发进度。

- [ ] **清空缓存操作**
    *   **模块**: gateway 模块 + 前端
    *   **说明**: 平台侧手动清空网关缓存、网关离线时指令缓存上线后执行。

- [ ] **补发速率配置**
    *   **模块**: 系统设置页面
    *   **说明**: 补发速率 1-500 条/秒、修改后立即生效。

### Phase 2.3: 历史数据查询 (Historical Data)

- [x] **点位历史数据查询**
    *   **模块**: device-data 模块 + 前端 DataTrendChart
    *   **说明**: 按点位 + 时间范围查询、分页支持、趋势图展示（recharts）、前后端 API 已打通。

- [ ] **多点位对比**
    *   **模块**: 前端 DataTrendChart
    *   **说明**: 同实例内多个点位数据叠加对比展示。

- [ ] **补发数据去重**
    *   **模块**: data-collection 服务
    *   **说明**: 收到 replay Topic 数据时检查是否已存在、已存在则跳过。

### Phase 2.4: 系统设置 (System Settings)

- [ ] **系统设置页面**
    *   **模块**: 前端 Settings 页面
    *   **说明**: 补发速率配置、缓存相关全局设置。
