## Node-RED 插件目录 (nodered-plugin/)

> **极简设计**：插件只需要部署心跳基础流，采集逻辑由后端生成 flow。

```
nodered-plugin/
├── README.md                # 插件说明
└── base-flow.json           # 基础心跳流 JSON（核心文件）
```

### 基础流结构 (base-flow.json)

```json
[
  {
    "id": "heartbeat-inject",
    "type": "inject",
    "repeat": "30",
    "wires": [["heartbeat-func"]]
  },
  {
    "id": "heartbeat-func",
    "type": "function",
    "func": "msg.payload = { gatewayId: '配置的网关ID', timestamp: Date.now() }; return msg;",
    "wires": [["heartbeat-mqtt"]]
  },
  {
    "id": "heartbeat-mqtt",
    "type": "mqtt out",
    "topic": "gateway/{gatewayId}/heartbeat",
    "broker": "emqx",
    "wires": []
  },
  {
    "id": "mqtt-broker",
    "type": "mqtt-broker",
    "broker": "emqx",
    "port": "1883"
  }
]
```

### 安装方式
1. 在 Node-RED 界面导入 `base-flow.json`
2. 修改 `heartbeat-func` 中的 `gatewayId`
3. 配置 MQTT Broker 连接到 EMQX

### 带缓存的采集流结构

> 当 `cacheEnabled = true` 时，后端生成的 flow 包含缓存逻辑

```json
[
  {
    "id": "collect-inject",
    "type": "inject",
    "repeat": "1",
    "wires": [["s7-read"]]
  },
  {
    "id": "s7-read",
    "type": "S7 in",
    "wires": [["cache-check"]]
  },
  {
    "id": "mqtt-out",
    "type": "mqtt out",
    "topic": "devices/{instanceId}/data",
    "broker": "mqtt-broker",
    "wires": []
  },
  {
    "id": "cache-check",
    "type": "function",
    "func": "// 检查 MQTT 连接状态\n// 如果连接正常，直接发送\n// 如果连接断开，写入 SQLite\n// 网络恢复后，从 SQLite 读取并重发",
    "wires": [["mqtt-out"], ["sqlite-write"]]
  },
  {
    "id": "sqlite-write",
    "type": "function",
    "func": "// 写入本地 SQLite\n// node-red-node-sqlite",
    "wires": []
  }
]
```

## 规范说明

### 命名约定

| 类型 | 约定 | 示例 |
|---|---|---|
| 文件名 | kebab-case | `device-model.service.ts` |
| React 组件 | PascalCase | `GatewayList.tsx` |
| TypeScript 类型 | PascalCase | `GatewayStatus` |
| 数据库表 | snake_case | `device_instance` |
| 数据库字段 | snake_case | `last_heartbeat` |
| 路由路径 | kebab-case | `/device-instances` |

### 模块导入顺序

1. Node.js 内置模块
2. 第三方依赖
3. 项目内部模块（按相对路径排序）
