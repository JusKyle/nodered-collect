## 2. 关键文件说明 (Key Files)
*   **package.json**: 项目依赖管理文件（前后端各自独立）。
*   **docker-compose.yml**: 一键启动 postgres/redis/emqx/nodered/backend/frontend 服务。
*   **backend/prisma/schema.prisma**: 数据库 Schema 定义，核心表包括 Gateway/DeviceModel/PointModel/DeviceInstance/SyncRecord。
*   **backend/.env.example**: 环境变量示例（禁止上传敏感信息，包含 DATABASE_URL/REDIS_URL/MQTT_HOST/JWT_SECRET 等）。
*   **nodered-plugin/base-flow.json**: Node-RED 心跳基础流，由平台下发到网关执行心跳上报。
*   **README.md**: 项目入口文档，必须包含"如何启动"说明（本地运行前后端、Docker 启动其他服务）。

## 3. 模块说明 (Module Description)
*   **backend/src/modules/**: 业务逻辑的核心，按 PRD 核心板块拆分。
    *   `gateway`: 网关管理（注册、心跳、状态监控、性能指标）
    *   `device-model`: 设备模板管理（创建、版本管理、点位定义）
    *   `device-instance`: 设备实例管理（创建、配置、网关分配、点位继承）
    *   `sync`: 配置下发与同步（版本比对、Flow 生成、Node-RED API 调用）
    *   `registration`: 网关注册（nodered-plugin 主动上报）
    *   `device-data`: 设备数据接收与存储（MQTT 订阅、时序库写入）
*   **backend/src/services/**: 公共服务，跨模块复用。
    *   `heartbeat.service.ts`: 心跳检测与状态判定
    *   `mqtt.service.ts`: MQTT 连接管理与消息订阅
    *   `sse.service.ts`: SSE 实时推送服务
    *   `dispatch.service.ts`: 配置下发引擎（生成 Flow、调用 Node-RED API）
    *   `data-collection.service.ts`: 数据采集与入库
    *   `token.service.ts`: Token 管理与验证
*   **frontend/src/pages/**: 页面组件，按功能模块组织。
    *   `gateway/`: 网关列表、详情、测试连接
    *   `device-model/`: 设备模板列表、点位编辑、版本管理
    *   `device-instance/`: 设备实例列表、详情、下发配置
    *   `sync/`: 下发历史记录
*   **frontend/src/stores/**: Zustand 状态管理，按模块分 Store。
*   **nodered-plugin/**: 极简设计，仅包含心跳基础流 JSON；采集逻辑由后端动态生成 Flow 并下发。

## 4. 文件放置规则 (Placement Rules)
*   **页面/路由**：放在 `frontend/src/pages/[对应模块]`，每个页面包含 List/Create/Edit/Detail/ConfirmBubble 等组件。
*   **业务逻辑**：放在 `backend/src/modules/[对应模块]`，遵循 Controller → Service → Repository 分层，禁止直接写在路由文件中。
*   **公共组件**：放在 `frontend/src/components`（如 DataTable/Sidebar/StatusBadge）。
*   **公共服务**：放在 `backend/src/services`（如 heartbeat/mqtt/sse/dispatch）。
*   **API 调用层**：放在 `frontend/src/api`，每个模块一个 API 文件（如 gateway.api.ts）。
*   **状态管理**：放在 `frontend/src/stores`，每个模块一个 Store 文件（如 gateway.store.ts）。
*   **数据库 Schema**：放在 `backend/prisma/schema.prisma`，禁止分散在模块内部。
*   **部署配置**：放在 `deployment/`，禁止放在 backend/frontend 根目录。
