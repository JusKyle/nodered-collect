# 开发规范

## 代码风格

### TypeScript

- 使用 TypeScript strict 模式
- 避免使用 `any`，优先使用具体类型
- 接口和类型别名使用 PascalCase
- 枚举成员使用 UPPER_SNAKE_CASE

```typescript
// ✅ 好的例子
interface GatewayDTO {
  id: string
  name: string
  status: GatewayStatus
}

// ❌ 避免
const data: any = fetchData()
```

### React 组件

- 使用函数组件 + Hooks
- Props 类型定义使用 interface
- 组件文件使用 PascalCase
- 每个组件不超过 300 行逻辑

```tsx
// ✅ 好的例子
interface GatewayListProps {
  onRefresh: () => void
}

export function GatewayList({ onRefresh }: GatewayListProps) {
  // ...
}
```

### 样式

- 使用 TailwindCSS 工具类
- 避免内联 style（除非动态值）
- 颜色使用 design token

## API 设计

### RESTful 规范

| 操作 | 方法 | 路径 |
|---|---|---|
| 列表 | GET | `/gateways` |
| 详情 | GET | `/gateways/:id` |
| 创建 | POST | `/gateways` |
| 更新 | PUT | `/gateways/:id` |
| 删除 | DELETE | `/gateways/:id` |

### 请求格式

```typescript
// 创建网关
POST /api/gateways
{
  "name": "Gateway 1",
  "address": "192.168.1.100",
  "port": 1880,
  "adminToken": "xxx"
}

// 更新网关
PUT /api/gateways/:id
{
  "name": "Gateway 1 Updated",
  "adminToken": "new-token"
}
```

### 响应格式

```typescript
// 成功响应
{
  "success": true,
  "data": { ... },
  "message": "操作成功"
}

// 错误响应
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid input"
  }
}
```

## 数据库规范

### Prisma 使用

- Schema 文件位于 `backend/prisma/schema.prisma`
- 使用 Prisma Migrate 管理迁移
- 不要直接操作数据库，使用 Repository 层

### 查询规范

```typescript
// ✅ 使用 Repository 层
const gateway = await repository.getGatewayById(id)

// ❌ 避免直接 Prisma 调用
const gateway = await prisma.gateway.findUnique({ where: { id } })
```

## 测试规范

### 单元测试

- 使用 Jest + ts-jest
- 测试文件放在 `__tests__/` 目录
- 测试覆盖核心业务逻辑

```typescript
// __tests__/gateway.service.test.ts
describe('GatewayService', () => {
  it('should create gateway', async () => {
    const gateway = await createGateway({
      name: 'Test',
      address: 'localhost',
      port: 1880
    })
    expect(gateway.name).toBe('Test')
  })
})
```

## 状态管理

### Zustand Store

- 每个业务模块一个 store
- Store 文件使用 `.store.ts` 后缀
- 同步状态到 localStorage 时注意序列化

```typescript
// stores/gateway.store.ts
interface GatewayStore {
  gateways: Gateway[]
  fetchGateways: () => Promise<void>
}

export const useGatewayStore = create<GatewayStore>((set) => ({
  gateways: [],
  fetchGateways: async () => {
    const gateways = await gatewayApi.getAllGateways()
    set({ gateways })
  }
}))
```

## 错误处理

### 后端错误

```typescript
// 使用 try-catch
try {
  const result = await someAsyncOperation()
  return res.success(result)
} catch (error) {
  return res.error(error.message, 'OPERATION_FAILED')
}
```

### 前端错误

```typescript
// API 调用错误处理
try {
  await gatewayApi.deleteGateway(id)
  toast.success('删除成功')
} catch (error) {
  toast.error(error.message || '删除失败')
}
```

## 安全规范

### 敏感信息

- Token 等敏感信息不记录日志
- 使用环境变量管理密钥
- 前端不存储明文密码

### 认证

- API 请求携带 JWT Token
- Token 过期时前端跳转登录页
- 敏感操作需要二次确认

## 性能优化

### 后端

- 使用 Redis 缓存频繁访问的数据
- 数据库查询添加索引
- 避免 N+1 查询

### 前端

- 组件懒加载
- 列表分页处理
- 避免不必要的重渲染

## 文档维护

### API 文档

- 使用 JSDoc 注释
- 接口变更时更新文档
- 标注废弃接口

### 代码注释

- 复杂逻辑添加注释
- 业务规则说明
- TODO 和 FIXME 标记
