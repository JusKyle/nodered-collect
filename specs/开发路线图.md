### 1.2 依赖关系说明

| 模块 | 依赖项 | 说明 |
|------|--------|------|
| 网关管理 | 基础设施 | 网关数据存储、心跳检测需要 Redis |
| 设备模型管理 | 基础设施 | 模型数据存储、点位配置 |
| 设备实例管理 | 设备模型 + 网关管理 | 实例从模型创建、需绑定网关 |
| 配置下发与同步 | 设备实例 + 网关管理 | 下发配置到网关、设备实例 |
| Node-RED 插件 | 后端 API | 注册、心跳、配置同步依赖后端服务 |

## 2. 里程碑规划

### Milestone 1: MVP（最小可行性产品）

**目标**: 完成基础设施和网关管理，实现网关注册、心跳检测、状态监控。

**范围**:
- 基础设施搭建（数据库、Redis、MQTT）
- 公共组件开发（中间件、工具函数）
- 网关管理完整功能
- 前端网关管理页面

**验收标准**: 网关可以注册、显示在线/离线状态、心跳正常工作。

---

### Milestone 2: 核心功能（设备管理闭环）

**目标**: 完成设备模型和设备实例管理，实现从模板创建设备、批量操作。

**范围**:
- 设备模型管理完整功能
- 设备实例管理完整功能
- 前端设备模型/实例页面
- 点位导入导出

**验收标准**: 可以创建设备模型、从模型创建设备实例、管理点位配置。

---

### Milestone 3: 配置下发（业务闭环）

**目标**: 完成配置下发引擎，实现设备配置下发到 Node-RED 网关。

**范围**:
- 配置下发引擎
- 同步状态管理
- 下发日志
- 前端同步记录页面

**验收标准**: 设备实例配置可以下发到网关、Node-RED 生成设备节点、状态同步正常。

---

### Milestone 4: 边缘端插件

**目标**: 完成 Node-RED 插件开发，实现边缘端完整功能。

**范围**:
- 设备管理器节点
- 设备实例节点
- 数据输出节点
- 注册服务
- 心跳服务
- 配置同步服务

**验收标准**: Node-RED 插件可以注册到平台、接收配置、上报数据。

---

### Milestone 5: 增强功能

**目标**: 完成断网处理、数据缓存补发、批量操作优化。

**范围**:
- 数据缓存服务（SQLite）
- 断网补发机制
- 本地微调回传
- 批量下发优化

**验收标准**: 断网后数据缓存、网络恢复后自动补发。

---

## 4. 开发顺序建议

### 4.1 推荐开发路径

```
Step 1: 基础设施搭建
  → 完成数据库、Redis、MQTT 配置
  → 完成公共中间件和工具函数

Step 2: 网关管理（后端 + 前端）
  → 完成网关 CRUD API
  → 完成心跳检测服务
  → 完成前端网关列表和弹窗

Step 3: 设备模型管理（后端 + 前端）
  → 完成设备模型 CRUD API
  → 完成点位管理
  → 完成前端模型列表和弹窗

Step 4: 设备实例管理（后端 + 前端）
  → 完成设备实例 CRUD API
  → 完成从模型创建实例
  → 完成前端实例列表和弹窗

Step 5: 配置下发（后端 + 前端）
  → 完成配置下发引擎
  → 完成同步状态管理
  → 完成前端同步记录页面

Step 6: Node-RED 插件
  → 完成插件节点开发
  → 完成注册、心跳、配置同步服务
  → 完成数据缓存服务

Step 7: 增强功能
  → 完成断网缓存补发
  → 完成批量操作优化
  → 完成本地微调回传
```
