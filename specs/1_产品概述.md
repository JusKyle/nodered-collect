# 产品概述

## 项目名称
Node-RED 分布式采集网关集中管理系统（collecting-system）

## 项目简介
分布式 Node-RED 采集网关集中管理系统，通过设备模板化和配置批量下发降低多网关管理成本。

## 核心价值
- 统一管理多个边缘网关（Node-RED 实例）
- 设备模板化，支持 Modbus、S7、OPC UA、MQTT 等协议
- 配置批量下发，一键部署到多个网关
- 实时状态监控与断网数据缓存

## 核心模块

### 1. 边缘网关管理
- Node-RED 实例添加与心跳检测
- 网关状态监控（在线/离线/Token失效）
- 网关心跳流自动部署（HTTP 直连下发）

### 2. 设备模型管理
- 设备类型模板定义
- 点位模型配置（寄存器地址、数据类型等）
- 版本管理与模板复制

### 3. 设备实例管理
- 从模板创建设备实例
- 点位继承与覆盖
- 网关分配与变更

### 4. 配置下发与同步
- 通过 HTTP `/flows` API 下发配置到网关
- 动态生成 Node-RED 采集节点（S7/modbus/OPC UA）
- 下发状态追踪与日志

### 5. 实时状态推送
- 后端通过 SSE 主动推送网关状态变化
- 前端实时更新，无轮询闪烁

### 6. 断网数据缓存
- 网关与平台断网时，本地缓存采集数据
- 网络恢复后自动补发缓存数据
- 支持多级缓存开关（系统/网关/设备）

## 断网缓存功能

### 缓存策略

| 级别 | 配置位置 | 优先级 | 说明 |
|------|---------|--------|------|
| 系统级 | 系统配置 | 最低 | 全局开关，三级默认均为关闭 |
| 网关级 | 网关管理 | 中等 | 按网关启用/禁用缓存 |
| 设备级 | 设备实例 | 最高 | 按需启用，优先于网关和系统级 |

### 缓存流程

```
1. 正常网络状态：
   采集数据 → MQTT 发送 → EMQX → 后端处理

2. 网络断开状态：
   采集数据 → MQTT 发送失败 → 写入网关本地缓存（每条记录）

3. 网络恢复状态：
   读取缓存 → 按时间先后顺序、按系统级速率限制补发 → 删除已发送记录
```

### 缓存配置继承规则

```
设备实例.cacheEnabled
      │
      ├─── 有值 → 使用设备实例的设置
      │
      └─── 无值 → 使用网关.cacheEnabled
                      │
                      ├─── 有值 → 使用网关的设置
                      │
                      └─── 无值 → 使用系统默认设置 (false)
```

### 缓存约束

- 每条采集数据都缓存，保留 15 天内的数据
- 同时受物理存储上限约束，达到上限时停止写入或覆盖最旧数据
- 网络恢复后严格按时间先后顺序补发（最早的数据先补发）
- 补发速率遵守系统级配置（每秒 N 条，可配置）
- 开关调整立即生效，无需重新下发配置

## 用户场景
工业物联网场景中，大量边缘网关部署在不同地点，需要集中管理配置和数据采集。

## 目标用户
- 工业自动化工程师
- IoT 系统运维人员
- 数据采集系统管理员
