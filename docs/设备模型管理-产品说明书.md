# 设备模型管理 — 产品需求说明书

## 1. 背景与价值

**为什么做**：在工业物联网场景中，同一型号的设备（如某品牌的 PLC）可能有数十台甚至上百台。如果每台设备都需要从零开始配置采集点位，工作量巨大且易出错。设备模型管理模块解决这个问题——通过定义设备类型模板和点位模型，一次定义、多处复用，大幅降低设备接入成本。

**做成什么样**：提供设备模型的创建、编辑、复制、导入点位等能力。每个模型包含设备基本信息（厂商、型号、协议）和点位模型列表（点位名称、地址、数据类型、采集频率等）。模型修改后保留版本历史，已下发的设备实例不会自动更新到新版本。

**关联**：产品概述 §3.2 模块二、§4 采集协议与点位模型、§9.2 验收标准

---

## 2. 范围

### 本次做

- 设备模型列表展示（名称、厂商、型号、协议、点位数量、状态、版本号、实例使用数）
- 创建设备模型（基本信息 + 点位模型）
- 编辑设备模型（修改基本信息或点位 → 新版本）
- 复制模型（基于现有模型快速复制创建新模型）
- 批量导入点位（CSV/Excel 导入，导入前预览）
- 模型状态管理（启用/停用，停用后不能用于创建设备实例）
- 模型版本管理（版本保留，向后兼容）
- 查看版本历史
- 删除模型（禁止删除已被设备实例使用的模型）
- 列表搜索（按名称）、筛选（按协议、按状态）

### 本次不做

- 模型模板市场（公开共享模型） — 内部使用场景，暂不需要
- 点位分组管理 — 当前点位数量不多，平铺展示即可
- 模型差异比对（版本间差异） — 版本保留已满足基本需求
- 模型导出 — 当前规模下手动配置可接受

## 4. 需求详述

### 4.1 核心流程

**主流程 A — 创建设备模型**

1. 运维人员在「设备模型」页面点击 `新建模型` 按钮，打开「新建设备模型」弹窗
2. 填写基本信息（名称、厂商、型号、协议）
3. 添加点位模型（逐个添加或批量导入）
4. 点击 `保存`，模型创建成功，状态为"启用"，版本号为 v1.0

**主流程 B — 编辑设备模型**

1. 运维人员在列表中找到目标模型，点击 `编辑`
2. 修改基本信息或点位（新增/编辑/删除点位）
3. 点击 `保存`，模型版本号递增（如 v1.0 → v1.1）
4. 已使用该模型的设备实例仍使用旧版本，不自动更新

**主流程 C — 批量导入点位**

1. 运维人员点击 `导入点位`，打开「导入点位」弹窗
2. 上传 CSV/Excel 文件
3. 系统解析文件，在弹窗中预览导入结果（成功/失败行）
4. 运维人员确认后点击 `导入`，点位添加到模型

### 4.2 交互规则

- **删除操作**：弹出二次确认气泡，文字"确定删除模型「XXX」吗？"
- **停用操作**：弹出确认气泡，文字"确定停用模型「XXX」吗？停用后将不能用于创建新的设备实例。"
- **必填校验**：表单提交时实时校验必填项，错误字段红框 + 下方红字提示
- **重复点击防抖**：保存按钮点击后置灰，请求返回后恢复
- **点位编码唯一**：同一模型内点位编码不可重复，重复时红框提示"该编码已存在"
- **导入预览**：上传文件后先显示预览列表，区分成功行和失败行，失败行标注原因

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 模型修改后版本号递增，已使用该模型的设备实例保留旧版本，不自动更新 |
| BR-002 | 禁止删除已被设备实例使用的模型，必须先解除所有引用 |
| BR-003 | 停用的模型不能用于创建新的设备实例，但已创建的实例不受影响 |
| BR-004 | 同一模型内点位编码必须唯一 |
| BR-005 | 采集频率范围为 100ms-3600000ms（100毫秒到1小时），默认 1000ms |
| BR-006 | 数据类型支持：int、float、string、bool、date、datetime、json |
| BR-007 | 读写权限支持：read、write、readWrite |
| BR-008 | 协议类型从预定义列表选择：Modbus TCP、S7、OPC UA、MQTT、HTTP、CoAP、MTConnect、FOCAS |

## 5. 功能需求

### 5.1 设备模型管理 - 模型列表页

#### 页面综述
- **功能描述**：管理所有设备类型模板，支持创建、编辑、复制、导入点位、状态切换、删除、查看版本历史
- **权限控制**：运维人员、设备管理人员可查看；仅运维人员可执行编辑、删除、新建、状态切换

#### 筛选/搜索区
- **搜索字段**：
  - `模型名称`：输入框，模糊匹配，默认空
  - `协议类型`：下拉选，选项 `全部` + 预定义协议列表，默认 `全部`
  - `状态`：下拉选，选项 `全部` / `启用` / `停用`，默认 `全部`
- **交互**：
  - 点击 `查询`：更新列表数据，重置页码为 1
  - 点击 `重置`：清空所有筛选条件
  - 切换下拉选：自动触发查询

#### 列表展示区
- **排序规则**：默认按 `创建时间` 倒序排列
- **列表字段**：
  - `模型名称`：文本，最多展示 1 行，超长省略号
  - `厂商`：文本
  - `型号`：文本
  - `协议`：文本，显示协议名称
  - `点位数量`：数字，格式"X 个点位"
  - `状态`：标签，绿色=启用，灰色=停用
  - `版本号`：文本，格式"vX.Y"
  - `实例使用数`：数字，格式"已使用 X 次"
  - `创建时间`：文本，格式"YYYY-MM-DD HH:mm"
- **批量操作**：
  - `批量停用`：勾选多个启用状态的模型后显示，点击弹出确认气泡

#### 操作列
- `编辑`：打开 `编辑设备模型（弹窗）`，回显当前数据
- `复制`：打开 `复制设备模型（弹窗）`，预填原模型数据
- `导入点位`：打开 `导入点位（弹窗）`
- `启用/停用`：切换模型状态
- `删除`：弹出二次确认气泡，确认后删除
- `版本历史`：打开 `版本历史（弹窗）`

#### 功能按钮
- `新建模型`：打开 `新建设备模型（弹窗）`

### 5.2 新建设备模型（弹窗）

#### 打开方式
点击列表页 `新建模型` 按钮

#### 基本信息区
- `模型名称`：必填，1-50 字符
- `厂商`：必填，1-50 字符
- `型号`：必填，1-50 字符
- `协议`：必填，下拉选，选项为预定义协议列表
- `描述`：选填，最多 200 字符

#### 点位列表区
- 初始状态：空列表，显示"暂无点位，请添加"
- 添加点位：点击 `添加点位` 按钮，在列表末尾新增一行
- 点位字段：
  - `点位名称`：必填，1-50 字符
  - `点位编码`：必填，1-30 字符，同一模型内唯一
  - `数据类型`：下拉选，选项 `int` / `float` / `string` / `bool` / `date` / `datetime` / `json`
  - `读写权限`：下拉选，选项 `read` / `write` / `readWrite`
  - `寄存器地址`：必填，1-100 字符
  - `数据偏移`：选填，整数，默认 0
  - `数据长度`：选填，整数，默认根据数据类型自动设置
  - `采集频率(ms)`：必填，整数，范围 100-3600000，默认 1000
  - `超时时间(ms)`：选填，整数，默认 3000
  - `单位`：选填，1-20 字符
  - `转换公式`：选填，支持简单算术表达式，如"value * 0.1 + 5"
  - `告警阈值(下限)`：选填，数字
  - `告警阈值(上限)`：选填，数字
- 删除点位：每行末尾 `删除` 按钮
- 导入点位：点击 `导入点位` 按钮，打开 `导入点位（弹窗）`

#### 按钮
- `取消`：关闭弹窗
- `保存`：保存模型

#### 校验规则
- 必填项缺失：字段下方红字提示"请输入 XXX"
- 点位编码重复：红框提示"该编码已存在"
- 采集频率超出范围：红字提示"采集频率需在 100-3600000ms 之间"

### 5.3 编辑设备模型（弹窗）

#### 打开方式
点击列表行 `编辑` 按钮

#### 基本信息区
同「新建设备模型」，所有字段均可修改

#### 点位列表区
同「新建设备模型」，支持新增、编辑、删除点位

#### 版本提示
- 弹窗顶部黄色提示条："修改将创建新版本，已使用该模型的设备实例不会自动更新"

#### 按钮
- `取消`：关闭弹窗
- `保存`：保存并创建新版本

### 5.4 复制设备模型（弹窗）

#### 打开方式
点击列表行 `复制` 按钮

#### 表单字段
- `模型名称`：必填，预填原模型名称 + "_副本"
- `厂商`：预填原模型值，可修改
- `型号`：预填原模型值，可修改
- `协议`：预填原模型值，可修改
- `描述`：预填原模型值，可修改

#### 点位处理
- 自动复制原模型的所有点位
- 复制后点位编码不变

#### 按钮
- `取消`：关闭弹窗
- `保存`：创建新模型，版本号为 v1.0

### 5.5 导入点位（弹窗）

#### 打开方式
点击编辑/新建弹窗中的 `导入点位` 按钮

#### 操作区
- 文件上传：`选择文件` 按钮，支持 CSV/Excel
- 模板下载：`下载导入模板` 按钮

#### 预览区
- 上传成功后显示预览列表：
  - 成功行：绿色标记，显示将导入的点位信息
  - 失败行：红色标记，显示失败原因（如"点位编码已存在"、"数据类型错误"）
- 统计："共 X 行，成功 X 行，失败 X 行"

#### 按钮
- `取消`：关闭弹窗
- `导入`：导入所有成功行的点位

#### 导入模板格式
| 点位名称 | 点位编码 | 数据类型 | 读写权限 | 寄存器地址 | 数据偏移 | 数据长度 | 采集频率(ms) | 超时时间(ms) | 单位 | 转换公式 | 告警阈值(下限) | 告警阈值(上限) |

### 5.6 版本历史（弹窗）

#### 打开方式
点击列表行 `版本历史` 按钮

#### 列表展示
- 字段：`版本号`、`更新时间`、`更新人`、`变更说明`
- 排序：按版本号倒序

#### 操作
- 查看详情：点击版本号，展开显示该版本的完整点位列表

### 5.7 删除确认（气泡）

#### 打开方式
点击列表行 `删除` 按钮

#### 提示文案
```
确定删除模型「XXX」吗？
[取消] [确定]
```

#### 特殊提示
- 若模型已被设备实例使用：气泡中额外显示红色提示"该模型已被 X 个设备实例使用，无法删除"，`确定` 按钮置灰

## 6. 验收标准

### 6.1 正常流程 (Happy Path)

- [ ] **AC-001**: 创建设备模型 - 基本信息填写
  - **Given** 运维人员在「新建设备模型」弹窗中填写了名称、厂商、型号、协议
  - **When** 点击 `保存`
  - **Then** 弹窗关闭，模型列表新增一行，状态显示为"启用"，版本号为"v1.0"

- [ ] **AC-002**: 创建设备模型 - 添加点位
  - **Given** 运维人员在「新建设备模型」弹窗中填写了基本信息
  - **When** 点击 `添加点位`，填写点位名称、编码、数据类型、寄存器地址等信息后点击 `保存`
  - **Then** 模型创建成功，列表中该模型的点位数量显示为 1

- [ ] **AC-003**: 创建设备模型 - 多个点位
  - **Given** 运维人员在「新建设备模型」弹窗中添加了 5 个点位
  - **When** 点击 `保存`
  - **Then** 模型创建成功，列表中该模型的点位数量显示为 5

- [ ] **AC-004**: 批量导入点位 - 上传文件
  - **Given** 运维人员在「导入点位」弹窗中选择了包含点位数据的 CSV 文件
  - **When** 点击 `打开`
  - **Then** 弹窗中显示预览列表，所有行标记为成功或失败

- [ ] **AC-005**: 批量导入点位 - 导入成功
  - **Given** 预览列表中有 3 行成功、0 行失败
  - **When** 点击 `导入`
  - **Then** 弹窗关闭，编辑/新建弹窗的点位列表中新增 3 个点位

- [ ] **AC-006**: 复制设备模型
  - **Given** 列表中存在模型"西门子S7-1200"，包含 10 个点位
  - **When** 点击 `复制`，修改名称为"西门子S7-1200_副本"后点击 `保存`
  - **Then** 列表中新增模型"西门子S7-1200_副本"，版本号为"v1.0"，点位数量为 10

- [ ] **AC-007**: 查看版本历史
  - **Given** 模型已多次修改，存在 v1.0、v1.1、v1.2 三个版本
  - **When** 点击 `版本历史`
  - **Then** 弹窗中显示版本列表，按版本号倒序排列，包含版本号、更新时间、更新人

- [ ] **AC-008**: 停用模型
  - **Given** 模型状态为"启用"
  - **When** 点击 `停用`，在确认气泡中点击 `确定`
  - **Then** 模型状态变为"停用"，列表中状态标签变为灰色

- [ ] **AC-009**: 启用模型
  - **Given** 模型状态为"停用"
  - **When** 点击 `启用`
  - **Then** 模型状态变为"启用"，列表中状态标签变为绿色

- [ ] **AC-010**: 按名称搜索
  - **Given** 列表中有多个模型
  - **When** 运维人员在搜索框输入关键字
  - **Then** 列表实时过滤，仅显示名称包含该关键字的模型

- [ ] **AC-011**: 按协议筛选
  - **Given** 列表中有 Modbus TCP 和 S7 协议的模型
  - **When** 运维人员将协议筛选切换为"Modbus TCP"
  - **Then** 列表仅显示 Modbus TCP 协议的模型

- [ ] **AC-012**: 编辑模型 - 版本递增
  - **Given** 模型当前版本为 v1.0
  - **When** 运维人员编辑模型并保存
  - **Then** 模型版本号变为 v1.1，列表中版本号更新

- [ ] **AC-013**: 删除模型
  - **Given** 模型未被任何设备实例使用
  - **When** 运维人员点击 `删除`，二次确认后点击 `确定`
  - **Then** 模型从列表中消失

### 6.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-014**: 表单必填项缺失
  - **Given** 运维人员点击 `保存` 创建模型
  - **When** 名称字段为空
  - **Then** 名称字段红框高亮，下方红字提示"请输入模型名称"，不关闭弹窗

- [ ] **AC-015**: 点位编码重复
  - **Given** 模型中已存在编码为"temp_001"的点位
  - **When** 运维人员添加新点位并填写相同编码
  - **Then** 编码字段红框高亮，下方红字提示"该编码已存在"

- [ ] **AC-016**: 采集频率超出范围
  - **Given** 运维人员在点位中填写采集频率为 50ms
  - **When** 失焦或点击 `保存`
  - **Then** 采集频率字段红框，下方红字提示"采集频率需在 100-3600000ms 之间"

- [ ] **AC-017**: 导入点位失败 - 编码重复
  - **Given** 模型中已存在编码为"temp_001"的点位
  - **When** 运维人员导入包含相同编码的 CSV 文件
  - **Then** 预览列表中该行标记为失败，显示红色提示"点位编码已存在"

- [ ] **AC-018**: 导入点位失败 - 数据类型错误
  - **Given** CSV 文件中数据类型列填写了"invalid_type"
  - **When** 运维人员上传文件
  - **Then** 预览列表中该行标记为失败，显示红色提示"数据类型错误"

- [ ] **AC-019**: 删除被使用的模型
  - **Given** 模型已被 3 个设备实例使用
  - **When** 运维人员点击 `删除`
  - **Then** 气泡中显示红色提示"该模型已被 3 个设备实例使用，无法删除"，`确定` 按钮置灰

- [ ] **AC-020**: 停用被使用的模型
  - **Given** 模型已被 5 个设备实例使用
  - **When** 运维人员点击 `停用`，确认后点击 `确定`
  - **Then** 模型状态变为"停用"，已创建的设备实例不受影响

- [ ] **AC-021**: 复制后点位编码不变
  - **Given** 原模型有点位编码"temp_001"
  - **When** 复制模型
  - **Then** 新模型中存在相同编码的点位

- [ ] **AC-022**: 导入模板下载
  - **Given** 运维人员在「导入点位」弹窗中
  - **When** 点击 `下载导入模板`
  - **Then** 浏览器下载 CSV 模板文件，包含所有点位字段列

### 6.3 业务规则验证 (Business Rules)

- [ ] **AC-023**: BR-001 模型版本保留（向后兼容）
  - **Given** 设备实例已使用模型 v1.0 创建，模型后续更新到 v1.1
  - **When** 查看该设备实例的点位配置
  - **Then** 设备实例仍使用 v1.0 的点位配置，不受 v1.1 变更影响

- [ ] **AC-024**: BR-002 禁止删除已使用的模型
  - **Given** 模型已被至少一个设备实例使用
  - **When** 运维人员尝试删除该模型
  - **Then** 删除操作被阻止，显示明确提示"该模型已被使用，无法删除"

- [ ] **AC-025**: BR-003 停用模型不可用于创建新实例
  - **Given** 模型状态为"停用"
  - **When** 运维人员在「设备实例」页面尝试从该模型创建设备实例
  - **Then** 下拉列表中该模型不可选（置灰）或不显示

- [ ] **AC-026**: BR-004 点位编码唯一性
  - **Given** 模型中已存在编码为"press_001"的点位
  - **When** 运维人员添加编码相同的新点位
  - **Then** 系统拒绝保存，提示"该编码已存在"

- [ ] **AC-027**: BR-005 采集频率范围
  - **Given** 运维人员在点位中填写采集频率
  - **When** 输入值为 99ms 或 3600001ms
  - **Then** 系统拒绝保存，提示"采集频率需在 100-3600000ms 之间"
