# 边缘网关管理 — 产品需求说明书

## 1. 背景与价值

**为什么做**：在多厂区、多设备的工业物联网场景中，每个厂区都部署了独立的 Node-RED 边缘网关。运维人员需要管理 10+ 个网关时，传统的「逐个登录 Node-RED 编辑器」方式效率低、易出错、配置难以追溯。本模块是整个系统的入口，负责把分散的网关纳管到统一平台。

**做成什么样**：提供一个集中的网关管理页面，支持两种注册方式（平台主动添加、网关主动注册），实时显示每个网关的在线状态、版本信息、运行状态和设备数量，让运维人员对全网网关一目了然。

**关联**：产品概述 §3.2 模块一、§7 网关断网处理、§9.1 验收标准

---

## 2. 范围

### 本次做

- 网关列表展示（在线状态、版本、运行状态、设备数量、地址端口）
- 网关注册（平台主动添加、网关主动注册两种方式）
- 网关信息编辑（名称、地址、端口、Admin Token）
- 网关删除（已下发设备转为"未绑定"）
- 测试连接（验证 Admin Token 有效性）
- 心跳检测（网关 → 平台 推送，30s/次，90s 无响应判定离线）
- 状态自动恢复（下次心跳到达即变在线）
- Token 失效的明显提示
- 列表搜索（按名称）、筛选（按状态）、自动刷新

### 本次不做

- 网关分组 / 标签管理 — 设备数量不大，暂不需要分组
- 告警通知（离线告警推送） — 已在产品概述 §11.2 列为后期扩展
- 网关批量导入 / 导出 — 当前规模下手动添加可接受
- 操作审计日志 — 已在产品概述 §11.2 列为后期扩展
- 多租户隔离 — 已在产品概述 §11.2 列为后期扩展

## 4. 需求详述

### 4.1 核心流程

**主流程 A — 平台主动添加网关**

1. 运维人员在「边缘网关」页面点击 `新建网关` 按钮，打开「新建网关」弹窗
2. 填写 `网关名称`、`地址`、`端口`、`Admin Token` 等信息
3. 点击 `测试连接` 按钮，平台调用 Node-RED API 验证 Token 有效性
4. 校验通过后点击 `确认`，网关创建成功
5. 列表中新增一行，状态显示为"在线"或"离线"

**主流程 B — 网关主动注册**

1. 运维人员点击 `生成注册码` 按钮，平台生成一次性注册码（含有效期）
2. 运维人员将注册码填入 Node-RED 侧插件
3. Node-RED 使用注册码调用平台 API 完成绑定
4. 平台侧列表出现新网关记录，状态显示为"在线"

**主流程 C — 网关运维操作**

1. 运维人员在列表中找到目标网关
2. 点击 `测试连接` → 验证 Token 仍然有效
3. 点击 `编辑` → 修改名称/地址/端口/Token
4. 点击 `查看已下发设备` → 跳转该网关的设备列表
5. 点击 `删除` → 二次确认 → 设备转为"未绑定"状态

### 4.2 交互规则

- **测试连接失败**：弹窗中红色提示失败原因（如"Token 无效"、"无法连接到目标地址"），不关闭弹窗
- **删除操作**：弹出二次确认气泡，文字"确定删除网关「XXX」吗？该网关下 N 个设备将转为未绑定状态"
- **Token 显示**：Admin Token 输入框默认脱敏（密码模式），提供"显示/隐藏"切换按钮
- **必填校验**：表单提交时实时校验必填项，错误字段红框 + 下方红字提示
- **重复点击防抖**：保存按钮点击后置灰，请求返回后恢复
- **列表自动刷新**：状态、最后在线时间等字段每 5s 自动刷新（用户可暂停）
- **离线网关视觉提示**：离线网关在列表中行底色变浅灰，状态点为灰色

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 网关名称允许重名，平台以内部 ID 唯一标识 |
| BR-002 | 心跳方向为「网关 → 平台推送」，间隔 30s |
| BR-003 | 平台连续 3 次未收到心跳（90s）则判定为离线 |
| BR-004 | 平台收到下次心跳即自动恢复为在线，无需人工操作 |
| BR-005 | Admin Token 失效时，下发/心跳/测试连接操作均返回 401，平台需明显提示用户重新填写 |
| BR-006 | 删除网关时，其下已下发设备的网关绑定关系解除，设备状态变更为"未绑定"，但设备本身不删除 |
| BR-007 | 注册码有效期 10 分钟，过期失效需重新生成 |
| BR-008 | 端口号范围 1-65535 |

## 5. 功能需求

### 5.1 边缘网关管理 - 网关列表页

#### 页面综述
- **功能描述**：管理所有接入平台的 Node-RED 网关，支持注册、编辑、删除、测试连接、查看已下发设备
- **权限控制**：运维人员、设备管理人员可查看；仅运维人员可执行编辑、删除、新建

#### 筛选/搜索区
- **搜索字段**：
  - `网关名称`：输入框，模糊匹配，默认空
  - `状态`：下拉选，选项 `全部` / `在线` / `离线`，默认 `全部`
- **交互**：
  - 点击 `查询`：更新列表数据，重置页码为 1
  - 点击 `重置`：清空所有筛选条件
  - 切换 `状态` 下拉选：自动触发查询

#### 列表展示区
- **排序规则**：默认按 `创建时间` 倒序排列
- **列表字段**：
  - `网关名称`：文本，最多展示 1 行，超长省略号
  - `状态`：使用状态点 + 文字，绿色=在线，灰色=离线
  - `最后在线时间`：文本，离线时显示"最后在线：YYYY-MM-DD HH:mm:ss"
  - `地址：端口`：文本，IP:Port 格式
  - `设备数量`：文本，格式"已下发 X / 运行中 Y"
  - `Node-RED 版本`：文本，如 "v3.1.0"，离线时显示"-"
  - `运行状态`：标签，正常/异常，离线时显示"离线"
- **自动刷新**：状态、最后在线时间、运行状态、已下发/运行中数量每 5s 自动刷新；提供 `暂停自动刷新` / `恢复自动刷新` 切换按钮

#### 操作列
- `测试连接`：调用 Node-RED API 验证 Token，结果用 Message 提示（成功绿色"连接成功"，失败红色显示原因）
- `编辑`：打开 `编辑网关（弹窗）`，回显当前数据
- `查看已下发设备`：跳转至该网关的设备实例列表页
- `删除`：弹出二次确认气泡，确认后删除

#### 功能按钮
- `新建网关`：打开 `新建网关（弹窗）`
- `生成注册码`：打开 `生成注册码（弹窗）`

### 5.2 新建网关（弹窗）

#### 打开方式
点击列表页 `新建网关` 按钮

#### 表单字段
- `网关名称`：必填，1-50 字符，允许重名
- `地址`：必填，IP 或域名格式校验
- `端口`：必填，整数 1-65535，默认 1880
- `Admin Token`：必填，密码模式输入，提供"显示/隐藏"按钮
- `描述`：选填，最多 200 字符

#### 按钮
- `测试连接`：调用 Node-RED API 验证 Token + 网络连通性
- `取消`：关闭弹窗
- `确认`：保存网关

#### 校验规则
- 必填项缺失：字段下方红字提示"请输入 XXX"
- 端口超出范围：红字提示"端口号需在 1-65535 之间"
- 地址格式错误：红字提示"请输入合法的 IP 或域名"

### 5.3 编辑网关（弹窗）

#### 打开方式
点击列表行 `编辑` 按钮

#### 表单字段
同「新建网关」，所有字段均可修改

#### 按钮
- `测试连接`：使用更新后的信息测试
- `取消`：关闭弹窗
- `确认`：保存修改

#### 提示
- 页面顶部黄色提示条："修改 Token 后需要等待下次心跳或手动 `测试连接` 验证"

### 5.4 生成注册码（弹窗）

#### 打开方式
点击列表页 `生成注册码` 按钮

#### 表单字段
- `网关名称`：必填
- `过期时间`：默认 10 分钟，可调整为 5/10/30 分钟

#### 展示内容
- `注册码`：注册码 + `复制` 按钮
- `过期时间`：显示具体到期时刻
- `使用说明`：文案"将此注册码填入 Node-RED 侧插件完成注册"

#### 状态
- 注册码过期时显示"已过期"灰色提示，禁用复制按钮
- 网关使用注册码完成注册后，弹窗自动关闭，列表出现新网关

### 5.5 删除确认（气泡）

#### 打开方式
点击列表行 `删除` 按钮

#### 提示文案
```
确定删除网关「XXX」吗？
该网关下 N 个设备将转为未绑定状态。
[取消] [确定]
```

## 6. 验收标准

### 6.1 正常流程 (Happy Path)

- [ ] **AC-001**: 平台主动添加网关 - 测试连接成功
  - **Given** 运维人员在「新建网关」弹窗中填写了合法的名称、地址、端口、Admin Token
  - **When** 点击 `测试连接` 按钮
  - **Then** 弹窗下方显示绿色"连接成功"提示，Token 字段右侧显示绿色对勾图标

- [ ] **AC-002**: 平台主动添加网关 - 创建成功
  - **Given** 运维人员已通过测试连接验证
  - **When** 点击 `确认` 按钮
  - **Then** 弹窗关闭，网关列表新增一行，状态显示为"在线"（或"离线"取决于是否在测试连接后已建立心跳）

- [ ] **AC-003**: 平台生成注册码
  - **Given** 运维人员在「生成注册码」弹窗中填写了网关名称
  - **When** 点击 `生成`
  - **Then** 弹窗内显示一串注册码（含 `复制` 按钮）和具体的过期时间

- [ ] **AC-004**: 网关主动注册成功
  - **Given** 平台已生成有效注册码
  - **When** Node-RED 侧使用该注册码完成注册
  - **Then** 平台侧「生成注册码」弹窗自动关闭，网关列表新增一行，状态显示为"在线"

- [ ] **AC-005**: 列表展示完整字段
  - **Given** 平台已有 1 个或多个网关
  - **When** 运维人员打开「边缘网关」列表页
  - **Then** 每个网关显示：名称、状态、最后在线时间、地址:端口、设备数量、Node-RED 版本、运行状态

- [ ] **AC-006**: 按名称搜索
  - **Given** 列表中有多个网关
  - **When** 运维人员在搜索框输入关键字
  - **Then** 列表实时过滤，仅显示名称包含该关键字的网关

- [ ] **AC-007**: 按状态筛选
  - **Given** 列表中有在线和离线的网关
  - **When** 运维人员将状态筛选切换为"离线"
  - **Then** 列表仅显示离线状态的网关

- [ ] **AC-008**: 列表自动刷新
  - **Given** 列表页已打开
  - **When** 网关状态发生变化（如离线 → 在线）
  - **Then** 5s 内列表中该网关的状态、最后在线时间等字段自动更新

- [ ] **AC-009**: 测试连接
  - **Given** 列表中存在任一网关
  - **When** 运维人员点击该网关的 `测试连接` 按钮
  - **Then** 顶部 Message 提示"连接成功"（绿色）或显示具体失败原因（红色）

- [ ] **AC-010**: 编辑网关
  - **Given** 网关已存在
  - **When** 运维人员点击 `编辑`，修改地址或端口后点击 `确认`
  - **Then** 弹窗关闭，列表中该网关的地址:端口字段更新为新值

- [ ] **AC-011**: 删除网关
  - **Given** 网关已存在（无论是否有关联设备）
  - **When** 运维人员点击 `删除`，二次确认后点击 `确定`
  - **Then** 网关从列表中消失

- [ ] **AC-012**: 查看已下发设备
  - **Given** 网关存在
  - **When** 运维人员点击 `查看已下发设备`
  - **Then** 页面跳转至该网关的设备实例列表页

- [ ] **AC-013**: 在线状态显示
  - **Given** 网关与平台心跳正常
  - **When** 运维人员查看列表
  - **Then** 该网关状态显示为绿色"在线"

- [ ] **AC-014**: 状态自动恢复
  - **Given** 网关之前被判定为离线
  - **When** 平台收到该网关的下一次心跳
  - **Then** 网关状态自动从"离线"变为"在线"，最后离线时间清空

### 6.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-015**: 测试连接失败 - Token 无效
  - **Given** 运维人员在「新建网关」/「编辑网关」弹窗中填写了地址和端口，但 Admin Token 错误
  - **When** 点击 `测试连接`
  - **Then** 弹窗下方显示红色提示"Token 无效或已过期"，不关闭弹窗

- [ ] **AC-016**: 测试连接失败 - 网络不通
  - **Given** 运维人员填写的地址/端口无法连通
  - **When** 点击 `测试连接`
  - **Then** 弹窗下方显示红色提示"无法连接到目标地址，请检查地址和端口"

- [ ] **AC-017**: 表单必填项缺失
  - **Given** 运维人员点击 `确认` 保存网关
  - **When** 任一必填字段为空
  - **Then** 第一个缺失字段红框高亮，下方红字提示"请输入 XXX"，不关闭弹窗

- [ ] **AC-018**: 端口格式错误
  - **Given** 运维人员填写了非数字或超范围的端口
  - **When** 失焦或点击 `确认`
  - **Then** 端口字段红框，下方红字提示"端口号需在 1-65535 之间"

- [ ] **AC-019**: 心跳超时判定离线
  - **Given** 网关之前状态为"在线"
  - **When** 平台连续 90s 未收到该网关的心跳（即连续 3 次未收到 30s 间隔的心跳）
  - **Then** 网关状态自动变为"离线"，行底色变浅灰，状态点为灰色

- [ ] **AC-020**: 删除有设备的网关
  - **Given** 网关下已下发 5 个设备实例
  - **When** 运维人员删除该网关并二次确认
  - **Then** 网关从列表消失；进入「设备实例」列表后，这 5 个设备的状态变更为"未绑定"，但设备记录本身仍存在

- [ ] **AC-021**: Token 失效明显提示
  - **Given** 网关已注册且使用过有效 Token
  - **When** Node-RED 侧修改了 Admin Token（导致原 Token 失效）
  - **Then** 列表中该网关状态保持最后一次心跳结果不变；当下一次心跳调用平台 API 返回 401 时，网关在列表中显示明显红色提示"Token 已失效，请更新"，并提供 `更新 Token` 快捷入口

- [ ] **AC-022**: 注册码过期
  - **Given** 平台已生成注册码超过 10 分钟
  - **When** Node-RED 侧使用该注册码尝试注册
  - **Then** 平台返回"注册码已过期"错误，Node-RED 侧需重新生成注册码

- [ ] **AC-023**: 网络抖动导致的临时离线
  - **Given** 网关短暂网络中断后又恢复
  - **When** 中断时间小于 90s
  - **Then** 网关状态保持"在线"不变（不出现中间"离线"状态）

### 6.3 业务规则验证 (Business Rules)

- [ ] **AC-024**: BR-001 网关名称允许重名
  - **Given** 平台已存在名称为"厂区A"的网关
  - **When** 运维人员新建另一个名称同样为"厂区A"的网关
  - **Then** 创建成功，列表中存在两条"厂区A"记录，但内部 ID 不同

- [ ] **AC-025**: BR-002/BR-003 心跳超时判定时长
  - **Given** 网关正常运行
  - **When** 网关停止推送心跳达到 90s
  - **Then** 网关状态变为"离线"（验证：第 60s 不变，第 90s 变）

- [ ] **AC-026**: BR-005 Token 失效提示明显
  - **Given** 网关已注册
  - **When** 平台调用 Node-RED /flows 或心跳推送返回 401
  - **Then** 平台不静默失败，网关在列表中以红色"Token 失效"标记，与普通离线状态视觉上明显区分

- [ ] **AC-027**: BR-006 删除网关不影响设备数据
  - **Given** 网关下有 N 个设备实例
