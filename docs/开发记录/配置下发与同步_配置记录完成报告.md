## 问题与决策

### API 路径沿用现有 sync 模块

**情况**：任务规划写的是 `/api/deployments` 与 `deployment` 模块，但当前代码和技术方案已有 `/api/sync/records` 与 `sync` 模块。
**处理**：沿用现有 `/api/sync/records`，避免新增平行模块造成重复。
**影响**：后续配置记录相关前端应继续使用 `sync` API，除非统一迁移到 deployment 模块。

### Flow 快照字段兼容

**情况**：真实历史记录把 Flow 存在 `payload.nodes`，技术方案描述为 `flowConfig`。
**处理**：详情服务兼容 `payload.flowConfig`、`payload.flows`、`payload.nodes`，统一输出 `flowConfig` 和 `flowNodes`。
**影响**：后续下发逻辑可以逐步统一写入 `payload.flowConfig`。
