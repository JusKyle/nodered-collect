# 配置下发与同步 — 产品需求说明书

## 1. 背景与价值

**为什么做**：设备实例创建完成后，需要将配置下发到边缘网关才能真正开始采集。同时，采集的数据需要上报到平台进行集中管理。在工业现场，网络中断是常态，必须确保断网期间数据不丢失。配置下发与同步模块是连接"平台管理"和"边缘采集"的关键桥梁。

**做成什么样**：平台通过 HTTP API 将设备配置下发到 Node-RED 网关，网关自动生成设备节点开始采集。采集数据通过 MQTT 上报到平台。断网期间数据本地 SQLite 缓存，网络恢复后自动补发。支持云端统一下发和 Node-RED 本地微调双向同步。

**关联**：产品概述 §3.2 模块四、§5 配置下发方案、§7 网关断网处理、§9.4-9.6 验收标准

## 2. 范围

### 本次做

- HTTP API 配置下发（调用 Node-RED /admin/flows）
- MQTT 数据上报（采集数据上报到平台）
- 下发自动重试（失败后最多重试 3 次）
- 下发日志记录（时间、内容、结果）
- 断网本地缓存（SQLite）
- 网络恢复自动补发（按时间顺序，带原始时间戳）
- 数据补发明示进度
- 缓存数据补发后自动删除
- 设备实例修改后自动重新下发配置
- 混合配置模式（云端下发 + 本地微调回传）
- Node-RED 手动绑定设备实例

### 本次不做

- 数据大屏展示（独立模块）
- 告警规则与通知（后期扩展）
- 历史数据分析与报表（后期扩展）
- 多协议数据转换（当前仅做采集和转发）

## 4. 需求详述

### 4.1 核心流程

**主流程 A — 配置下发**

1. 运维人员在平台点击设备实例的 `下发配置`
2. 平台调用 Node-RED `/admin/flows` API，携带设备配置（JSON格式）
3. Node-RED 接收配置，自动生成设备节点（配置节点、设备实例节点、协议采集节点、数据处理节点）
4. 节点部署完成，设备开始采集
5. 采集数据通过 MQTT 上报到平台
6. 平台更新实例状态为"运行中"

**主流程 B — 断网处理与数据补发**

1. 网关网络中断，平台判定网关离线
2. Node-RED 继续本地采集，数据写入 SQLite 缓存
3. 网络恢复，网关重新连接平台
4. SQLite 缓存数据按时间顺序补发，每条数据带原始时间戳
5. 平台接收补发数据，记录到数据库
6. 缓存数据补发成功后自动删除

**主流程 C — 混合配置模式**

1. 平台下发配置到网关（云端统一下发）
2. 运维人员在 Node-RED 中手动调整设备节点参数（本地微调）
3. 调整后的配置自动回传到平台，更新设备实例
4. 平台记录配置变更来源（"本地微调"标记）

### 4.2 交互规则

- **下发中状态**：点击 `下发配置` 后，按钮置灰显示"下发中..."，列表状态变为"下发中"
- **下发失败提示**：下发失败后，顶部红色提示失败原因，状态保持"待同步"
- **补发进度展示**：网络恢复后，网关详情页显示"正在补发 X 条数据"，进度条实时更新
- **日志查看**：点击 `查看日志` 打开下发日志弹窗，显示时间、操作、结果
- **本地微调回传**：Node-RED 中修改设备节点后，平台侧设备实例自动更新，显示"本地微调"标记

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 配置下发使用 HTTP API 调用 Node-RED `/admin/flows` |
| BR-002 | 采集数据通过 MQTT 上报到平台 |
| BR-003 | 下发失败后自动重试最多 3 次，间隔递增（10s → 30s → 60s） |
| BR-004 | 断网期间采集数据写入本地 SQLite 数据库缓存 |
| BR-005 | 网络恢复后自动补发缓存数据，按时间顺序，带原始时间戳 |
| BR-006 | 缓存数据补发成功后自动删除 |
| BR-007 | 设备实例修改后自动重新下发配置到网关 |
| BR-008 | Node-RED 中本地微调的配置可回传到平台 |
| BR-009 | Node-RED 中可手动选择绑定平台已下发的设备实例 |

## 5. 功能需求

### 5.1 配置下发管理 - 下发日志页面

#### 页面综述
- **功能描述**：查看配置下发的历史记录，包括下发时间、目标网关、设备实例、结果等
- **权限控制**：运维人员、设备管理人员可查看

#### 筛选/搜索区
- **搜索字段**：
  - `网关名称`：下拉选，选项 `全部` + 所有网关名称，默认 `全部`
  - `设备实例`：下拉选，选项 `全部` + 所有实例名称，默认 `全部`
  - `结果`：下拉选，选项 `全部` / `成功` / `失败`，默认 `全部`
  - `时间范围`：日期选择器，默认近 7 天
- **交互**：
  - 点击 `查询`：更新列表数据，重置页码为 1
  - 点击 `重置`：清空所有筛选条件

#### 列表展示区
- **排序规则**：默认按 `下发时间` 倒序排列
- **列表字段**：
  - `下发时间`：文本，格式"YYYY-MM-DD HH:mm:ss"
  - `目标网关`：文本，显示网关名称
  - `设备实例`：文本，显示实例名称
  - `操作类型`：标签，`下发` / `重新下发` / `解除下发`
  - `结果`：标签，绿色=成功，红色=失败
  - `重试次数`：数字，显示重试了几次
  - `备注`：文本，显示失败原因或成功提示

#### 操作列
- `查看详情`：打开 `下发详情（弹窗）`

### 5.2 下发详情（弹窗）

#### 打开方式
点击日志列表行 `查看详情` 按钮

#### 展示内容
- `下发时间`：精确到秒
- `目标网关`：网关名称 + 地址:端口
- `设备实例`：实例名称 + 设备地址
- `操作类型`：下发/重新下发/解除下发
- `下发内容`：JSON 格式，可折叠/展开
- `返回结果`：JSON 格式，可折叠/展开
- `重试记录`：时间、结果、原因
- `最终结果`：成功/失败

### 5.3 网关详情页 - 数据同步面板

#### 页面综述
- **功能描述**：在网关详情页中展示数据同步状态和补发进度
- **位置**：网关详情页右侧或下方

#### 同步状态区
- `网关状态`：在线/离线标签
- `数据上报状态`：正常/异常标签
- `MQTT 连接状态`：已连接/未连接标签
- `最近上报时间`：文本，格式"YYYY-MM-DD HH:mm:ss"
- `今日上报条数`：数字

#### 缓存与补发区
- `缓存数据条数`：数字（仅离线时有值）
- `补发进度`：进度条 + 文本"正在补发 X/Y 条"（仅恢复网络时有值）
- `补发状态`：未开始/补发中/已完成/部分失败

### 5.4 Node-RED 设备节点（混合配置模式）

#### 配置节点（Device Mgr）
- 存储设备实例元数据（设备 ID、名称、协议、连接信息）
- 运行时提供配置信息给采集节点

#### 设备实例节点
- 接收配置，实例化点位，管理采集生命周期
- 支持手动选择绑定平台已下发的设备实例

#### 协议采集节点
- 按协议采集数据（复用现有节点如 node-red-contrib-modbus）
- 支持参数微调（采集频率、超时时间等）
- 微调后自动回传到平台

#### 数据处理节点
- 单位转换、阈值检测、数据格式化
- 支持配置转换公式

#### MQTT Output 节点
- 将采集数据发送到平台 MQTT Broker
- 数据格式包含：设备 ID、点位编码、值、时间戳、质量标志

### 5.5 Node-RED 手动绑定设备实例（弹窗）

#### 打开方式
在 Node-RED 中添加设备实例节点时

#### 展示内容
- 下拉列表展示平台已下发到该网关的设备实例
- 每个选项显示：设备实例名称、所属模型、设备地址、点位数量

#### 操作
- 选择设备实例后自动带入点位配置
- 支持后期解绑和重新绑定

## 6. 验收标准

### 6.1 正常流程 (Happy Path)

- [ ] **AC-001**: 配置下发成功 - 单个实例
  - **Given** 设备实例状态为"待同步"，已绑定在线网关
  - **When** 运维人员点击 `下发配置`
  - **Then** 实例状态变为"运行中"，顶部显示绿色"下发成功"提示

- [ ] **AC-002**: 配置下发成功 - Node-RED 自动生成节点
  - **Given** 配置下发成功
  - **When** 运维人员登录 Node-RED 编辑器
  - **Then** 可看到自动生成的设备节点（配置节点、设备实例节点、协议采集节点、数据处理节点）

- [ ] **AC-003**: 查看下发日志
  - **Given** 平台已执行过多次配置下发
  - **When** 运维人员打开下发日志页面
  - **Then** 列表显示所有下发记录，包含时间、网关、实例、结果

- [ ] **AC-004**: 断网期间本地缓存
  - **Given** 网关正常运行，正在采集数据
  - **When** 网络中断，网关变为离线
  - **Then** Node-RED 继续本地采集，数据写入 SQLite 缓存

- [ ] **AC-005**: 网络恢复自动补发
  - **Given** 网关离线期间已缓存 100 条数据
  - **When** 网络恢复，网关变为在线
  - **Then** 缓存数据自动按时间顺序补发，每条数据带原始时间戳

- [ ] **AC-006**: 本地微调设备节点
  - **Given** 设备节点已在 Node-RED 中运行
  - **When** 运维人员在 Node-RED 中修改采集频率（从 1000ms 改为 2000ms）
  - **Then** 采集频率生效，调整后的配置回传到平台

- [ ] **AC-007**: 本地微调回传验证
  - **Given** Node-RED 中已修改采集频率并回传
  - **When** 运维人员在平台查看该设备实例的点位配置
  - **Then** 采集频率字段已更新为新值，显示"本地微调"标记

- [ ] **AC-008**: Node-RED 手动绑定设备实例
  - **Given** 平台已下发设备实例到网关
  - **When** 运维人员在 Node-RED 中添加设备实例节点，选择绑定该实例
  - **Then** 节点自动带入该实例的点位配置

- [ ] **AC-009**: 缓存数据补发后自动删除
  - **Given** 缓存数据已全部补发成功
  - **When** 查看网关 SQLite 缓存
  - **Then** 缓存记录为空

- [ ] **AC-010**: 数据上报到平台
  - **Given** 设备实例状态为"运行中"
  - **When** 查看平台数据接收状态
  - **Then** 平台持续收到该设备的采集数据

- [ ] **AC-011**: 设备实例修改后自动重新下发
  - **Given** 设备实例已下发且运行中
  - **When** 运维人员修改设备地址并保存
  - **Then** 平台自动重新下发配置到网关

- [ ] **AC-012**: 补发进度展示
  - **Given** 网关正在补发缓存数据
  - **When** 运维人员查看网关详情页
  - **Then** 页面显示补发进度条和"正在补发 X/Y 条"提示

### 6.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-013**: 下发失败 - 网关离线
  - **Given** 设备实例绑定的网关当前离线
  - **When** 点击 `下发配置`
  - **Then** 顶部红色提示"网关离线，无法下发配置"，状态保持"待同步"

- [ ] **AC-014**: 下发失败 - Admin Token 无效
  - **Given** 网关的 Admin Token 已失效
  - **When** 点击 `下发配置`
  - **Then** 下发失败，日志显示"Token 无效"，自动重试最多 3 次

- [ ] **AC-015**: 下发失败自动重试
  - **Given** 第一次下发失败（网络抖动）
  - **When** 等待重试
  - **Then** 系统自动重试，间隔递增（10s → 30s → 60s），最多重试 3 次

- [ ] **AC-016**: 断网期间持续采集
  - **Given** 网关离线
  - **When** 查看 Node-RED 运行状态
  - **Then** 采集节点继续运行，数据持续写入 SQLite

- [ ] **AC-017**: 断网时间过长 - 缓存溢出
  - **Given** 网关离线时间过长，缓存数据量达到上限
  - **When** 新数据产生
  - **Then** 最早的缓存数据被覆盖，系统记录缓存溢出日志

- [ ] **AC-018**: 补发失败 - 部分数据丢失
  - **Given** 缓存中有 100 条数据，其中 5 条补发失败
  - **When** 补发完成后
  - **Then** 95 条成功，5 条标记为失败；补发状态显示"部分失败"

- [ ] **AC-019**: 本地微调与平台下发冲突
  - **Given** Node-RED 中已微调采集频率，平台随后下发新配置
  - **When** 平台下发新配置
  - **Then** 平台配置覆盖本地微调，页面提示"已覆盖本地微调配置"

- [ ] **AC-020**: Node-RED 手动绑定不存在的实例
  - **Given** 平台已删除某设备实例
  - **When** 运维人员在 Node-RED 中尝试绑定该实例
  - **Then** 列表中不显示该实例，或提示"设备实例不存在"

- [ ] **AC-021**: MQTT 连接失败 - 自动重连
  - **Given** 平台 MQTT Broker 暂时不可用
  - **When** 查看网关数据上报状态
  - **Then** 数据暂时缓存，MQTT 自动重连，恢复后补发

- [ ] **AC-022**: 网络抖动导致临时离线
  - **Given** 网络短暂中断后立即恢复（小于 90s）
  - **When** 查看网关状态
  - **Then** 网关状态保持"在线"不变，不触发离线逻辑

### 6.3 业务规则验证 (Business Rules)

- [ ] **AC-023**: BR-001 HTTP API 下发
  - **Given** 平台执行下发操作
  - **When** 查看平台日志
  - **Then** 平台调用了 Node-RED `/admin/flows` API

- [ ] **AC-024**: BR-003 自动重试策略
  - **Given** 下发失败
  - **When** 监控系统行为
  - **Then** 系统在 10s、40s、100s 后分别重试（间隔递增）

- [ ] **AC-025**: BR-004 断网本地缓存
  - **Given** 网关离线
  - **When** 查看 SQLite 数据库
  - **Then** 采集数据已写入数据库

- [ ] **AC-026**: BR-005 补发带时间戳
  - **Given** 缓存数据已补发
  - **When** 查看平台接收到的数据
  - **Then** 每条数据都带有原始采集时间戳（非补发时间）

- [ ] **AC-027**: BR-006 补发后删除缓存
  - **Given** 缓存数据已全部补发成功
  - **When** 查看 SQLite 数据库
  - **Then** 缓存表为空

- [ ] **AC-028**: BR-007 设备修改自动重新下发
  - **Given** 设备实例已下发且运行中
  - **When** 修改设备实例信息并保存
  - **Then** 平台自动触发重新下发，无需手动操作

- [ ] **AC-029**: BR-008 本地微调回传
  - **Given** Node-RED 中修改了设备节点参数
  - **When** 等待同步
  - **Then** 平台侧设备实例参数自动更新

- [ ] **AC-030**: BR-009 Node-RED 手动绑定
  - **Given** 平台已下发设备实例到网关
  - **When** 在 Node-RED 中添加设备实例节点
  - **Then** 下拉列表中可看到该设备实例
